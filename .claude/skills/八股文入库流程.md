# Skill: 综合笔记整理流程

## 触发条件

当用户说以下任意话术时触发：
- "帮我整理笔记"
- "帮我入库"
- "整理一下"
- "整理今天的八股文"
- "把今天的八股文归档"
- "更新知识库"

---

## 第一阶段：Daily Note 格式重整

### Step 0: 读取并重构 Daily Note

读取当天的 `DailyNotes/YYYY-MM-DD.md`，按以下标准重构：

1. **补 frontmatter**：`tags: [daily, 日记]` + `date`
2. **按主题分块**：算法、项目、DevOps、知识调研等分别成独立小节
3. **填写完成情况**：根据正文内容判断哪些已完成
4. **提炼今日收获**：从正文中提取 3-5 条关键收获，不是简单复述，要提炼「下次能用的经验」
5. **补相关链接**：关联到已有的原子笔记和学习笔记
6. **补时间统计**：根据内容量合理估算（如用户未填则留空）
7. **清除模板残骸**：删掉未使用的占位符和空行

---

## 第二阶段：内容识别与分类

从 Daily Note 中识别以下几类内容，分别走不同处理路径：

| 内容类型 | 识别特征 | 处理方式 |
|---------|---------|---------|
| **八股文 Q&A** | 自问自答格式、有"问题"+"答案"结构 | → 第三阶段（八股文入库 SOP） |
| **八股文碎片** | 一句话技术点 + 解释，不够撑起独立笔记 | → 补入已有八股文（第四阶段） |
| **学习笔记** | 工具学习、调研、DevOps 等非面试 Q&A | → 第五阶段（学习笔记归档） |
| **算法刷题** | 力扣题号 + 思路 + 坑 | → 整理到 Daily Note，不单独建八股文 |
| **项目经验** | 上线、部署、排障等实战经历 | → 整理到 Daily Note + 关联待学清单 |

---

## 第三阶段：八股文入库 SOP（核心流程）

### Step 1: 读取源数据

提取 Daily Note 中「八股文自问自答」或类似板块的内容。

### Step 2: 内容分类

将每个八股问题按以下主题分类：

| 分类 | 关键词 |
|------|--------|
| **JVM** | JVM、GC、垃圾回收、内存模型、类加载、调优、OOM、StackOverflow |
| **Java 基础** | String、HashMap、集合、IO、NIO、反射、泛型、注解、异常、数组 |
| **Java 并发** | 线程池、AQS、锁、volatile、synchronized、CAS、ThreadLocal、JUC、并发 |
| **MySQL** | MySQL、索引、B+树、事务、隔离级别、锁、MVCC、redo log、binlog、慢查询、优化 |
| **Redis** | Redis、缓存、持久化、RDB、AOF、哨兵、集群、分布式锁、过期策略、淘汰策略 |
| **消息队列** | Kafka、RocketMQ、RabbitMQ、消息队列、Topic、Partition、Consumer、Producer、重复消费 |
| **Spring** | Spring、Spring Boot、Spring Cloud、IOC、AOP、Bean、自动装配、启动流程、事务 |
| **分布式** | 分布式、微服务、CAP、BASE、一致性、熔断、限流、降级、注册中心、网关、RPC、Dubbo |
| **网络** | TCP、UDP、HTTP、HTTPS、TCP/IP、三次握手、四次挥手、滑动窗口、拥塞控制、NIO、Netty |
| **操作系统** | Linux、进程、线程、内存、磁盘、IO、文件系统、进程间通信、调度 |
| **算法与数据结构** | 算法、排序、二叉树、链表、数组、哈希表、堆、栈、队列、时间复杂度、动态规划、贪心、双指针 |
| **设计模式** | 设计模式、单例、工厂、代理、观察者、策略、模板方法、责任链、装饰器 |
| **场景题** | 秒杀、短链、限流、排行榜、计数器、幂等、分布式ID、数据一致性、架构设计 |
| **排查与优化** | 排查、调优、性能、GC、CPU、内存、慢SQL、压测、监控、毛刺、RT、接口优化 |
| **AI / 大模型** | LLM、GPT、FunctionCall、MCP、Prompt、RAG、Agent、Embedding、向量、AI Coding、Cursor、Copilot、Claude Code |

分类规则：
- 一个问题可能涉及多个分类，以**最主要**的那个为准
- 如果无法确定分类，先放到「待分类」，并在 MOC 中标注

### Step 3: 创建原子笔记

为每个八股问题创建独立笔记：

**文件路径**：`知识库/八股文/{分类}/{问题标题}.md`（如 `八股文/Java基础/泛型与类型擦除.md`、`八股文/场景题/秒杀系统设计.md`）

**命名规范**：
- 去除编号（如 "1. "、"Q1. "）
- 保留核心疑问词（如 "为什么"、"如何"、"怎么"、"有什么区别"）
- 长度控制在 20 字以内

**跨 Day 去重规则**：
- 同一 topic 在多个日期出现时，保留**最新 + 最详细**的版本
- 不跨版本合并内容（避免笔记过长）
- 被跳过的版本在 MOC 中不重复记录

**笔记模板**：
```markdown
---
tags: [八股文, {分类}, 面试]
source: "[[DailyNotes/YYYY-MM-DD]]"
date: YYYY-MM-DD
---

# {问题标题}

## 问题
{原始问题描述}

## 答案
{详细答案内容}

## 项目关联
{如有实际项目经验，建立链接}

## 相关笔记
- [[相关笔记1]]
- [[相关笔记2]]

## 来源
原始记录：[[DailyNotes/YYYY-MM-DD#{锚点标题}]]
```

### Step 4: 更新 MOC 索引

读取 `知识库/MOC/八股文索引.md`，按分类将新问题添加到对应表格中。

**表格格式**：
```markdown
## {分类}

| 问题 | 来源 |
|------|------|
| [[原子笔记名\|显示标题]] | [[DailyNotes/YYYY-MM-DD\|MM-DD]] |
```

### Step 5: 更新 Daily Note

将 Daily Note 中八股文板块的详细内容替换为原子笔记链接。

**替换前**：
```markdown
#### 1. XXX 问题？
大量答案内容...
```

**替换后**：
```markdown
#### 1. XXX 问题？
详细答案见独立笔记：[[XXX 问题]]
```

---

## 第四阶段：八股文碎片 → 补入已有笔记

对于不够撑起独立笔记、但能增强已有八股文的技术点（如今天学到的「Arrays.copyOf 做数组深拷贝」），判断是否已有相关笔记：

1. 用关键词在 `知识库/八股文/{分类}/` 中搜索最相关的已有笔记
2. 在该笔记中添加一个新小节（如「### 数组的特殊情况」）
3. 保持原有内容不变，只追加新内容
4. 补充的来源标注为 `[[DailyNotes/YYYY-MM-DD]]`

**适用场景**：
- 刷题时踩的坑 → 补入相关的 Java 基础/算法笔记
- 线上排障经验 → 补入相关的排查与优化笔记
- 实战中验证了某个八股文知识点 → 在笔记末尾加「实战验证」小节

---

## 第五阶段：学习笔记归档

对于非八股文的学习内容（DevOps、工具、调研等），整理为独立学习笔记：

### 创建学习笔记

**文件路径**：`知识库/{问题标题}.md`（放在知识库根目录，不放八股文目录）

**笔记模板**：
```markdown
---
tags: [学习, {分类}]
date: YYYY-MM-DD
source: "[[DailyNotes/YYYY-MM-DD]]"
---

# {标题}

{结构化整理后的内容，含命令、代码、对比表等}

## 相关链接
- [[待学清单]] — 学习计划
- [[相关笔记]]
```

### 更新待学清单

在 `知识库/待学清单.md` 中：
1. 对应条目标记为 `[/]` 学习中，并添加 `→ [[笔记名]]` 链接
2. 在「来源追踪」表中补充发现记录

---

## 第六阶段：交叉链接验证

1. 检查今日新建/修改的笔记中所有 `[[...]]` 链接
2. 修复断链：目标不存在则删除或替换为已有笔记
3. 为今日笔记补充「相关笔记」反向链接（如果相关笔记也该引用今天的新笔记）

---

## 第七阶段：总结汇报

执行完成后，向用户汇报：

1. **Daily Note 整理**：格式重整了哪些板块
2. **八股文入库**：入库了几个问题，分别属于哪些分类，新建了哪些原子笔记
3. **已有笔记增强**：补入了哪些已有笔记的哪些内容
4. **学习笔记**：创建了哪些新的学习笔记
5. **待学清单更新**：哪些条目状态有变化
6. **待确认**：是否有无法分类的内容需要用户确认

---

## 项目经验型笔记

当用户提出以下需求时，创建**项目经验型**原子笔记（区别于标准自问自答式八股文）：

**触发话术**：
- "帮我写一个 XX 场景的八股文"
- "从项目经验角度总结 XX"
- "写一个面试用的项目经验笔记"
- "基于我的思路扩展 XX"

### 项目经验型笔记模板

```markdown
---
tags: [八股文, {分类}, 面试, 项目经验]
source: "[[MOC/八股文索引]]"
date: YYYY-MM-DD
---

# {问题标题}

## 问题
> {面试官原话}

## 回答策略
{一句话说清策略——锚定什么业务链路、覆盖哪些知识点}

## {核心链路}中的 {主题}

```
{ASCII 架构图}
```

## 逐层详细解释

### 1. {模式名}
**位置**：{在哪个环节}
> {为什么用}
```java
{实战代码}
```
**关键点**：{易错点}

## 面试回答话术模板
> {可直接口述的完整回答}

## 相关笔记
- [[相关笔记1]]

## 来源
基于项目经验总结，围绕「{核心链路}」串联{主题}。
```

---

## 注意事项

1. **绝不覆盖已有笔记**：已存在则跳过，只更新索引和链接
2. **尊重用户修改**：用户手动调整过的分类和命名以用户为准
3. **保持链接有效**：所有 `[[...]]` 链接的笔记必须真实存在
4. **分类不确定时**：先放入「待分类」，汇报中询问用户
5. **算法刷题不入库**：LeetCode 刷题记录一般不单独建八股文，除非内容足够撑起一篇 Q&A
6. **碎片补充优先**：一句话技术点优先考虑补入已有笔记，而非新建笔记
