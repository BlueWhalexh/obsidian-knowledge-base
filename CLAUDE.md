# Vault 知识库规范

## 📌 核心规则

### 「整理」的完整含义

当用户说"帮我整理笔记"、"整理一下"时，执行**综合整理**，包含以下全部步骤（不只是八股文入库）：

1. **Daily Note 格式重整** — 补充 frontmatter、结构化板块、清除模板残骸
2. **八股文提取** — 从日记中识别 Q&A 格式的八股内容，按 SOP 创建原子笔记 + 更新 MOC
3. **学习笔记归档** — 非八股文的学习内容（如 DevOps、工具、算法刷题）整理为独立学习笔记，关联到 [[待学清单]]
4. **交叉链接** — 检查今日内容是否能挂到已有笔记（如踩坑经验补入已有八股文），添加双向链接
5. **索引更新** — 更新相关的 MOC 索引页（八股文索引、场景题索引、待学清单）
6. **汇报** — 完成后向用户说明做了哪些改动

详见 `.claude/skills/八股文入库流程.md`。

## 📝 笔记规范

- **原子笔记**：一个知识点一篇，放在 `知识库/八股文/{分类}/` 下（如 `八股文/Java基础/`、`八股文/场景题/`）
- **命名**：去掉编号，保留疑问词，控制在 20 字内
- **frontmatter**：必须包含 `tags`, `source`, `date`
- **链接**：用 `[[wikilink]]`，不用纯路径
- **绝不覆盖已有笔记**：已存在则跳过创建，只更新索引和链接

## 📅 每日复盘习惯

用户每日复盘时通常是：
1. 在 DailyNotes 里写草稿
2. 让 Claude 帮忙整理成结构化笔记
3. 八股文内容入库到知识库

## ⚠️ 已知坑

- LeetCode 刷题记录一般**不**单独建八股文，除非内容足够撑起一篇 Q&A
- `Pasted image` 引用的是 Obsidian 截图附件，路径格式为 `../Attachments/xxx.png`
- vault 根目录绝对路径是 `D:\Document\Notes`，工具调用时用相对路径
