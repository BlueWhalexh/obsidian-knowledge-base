# 📚 Obsidian 个人知识库

基于 **LLM Wiki 方法论**（Karpathy 路线）构建的 Obsidian 知识管理系统。

## 🏗️ 架构设计

```
┌─────────────────────────────────────────────┐
│  raw/    源材料层    AI 只读不改             │
│  ├── DailyNotes/          每日日记草稿       │
│  └── 小记/                随手记录           │
├─────────────────────────────────────────────┤
│  wiki/   合成层      AI 维护的 Markdown      │
│  ├── 知识库/八股文/       原子化 Q&A 笔记    │
│  ├── 知识库/MOC/          索引页（Map of Content）│
│  └── 知识库/*.md          学习笔记           │
├─────────────────────────────────────────────┤
│  CLAUDE.md + .claude/skills/  规则层         │
│  ├── 笔记命名规范                           │
│  ├── 八股文入库 SOP                         │
│  └── 分类规则                               │
└─────────────────────────────────────────────┘
```

## 📂 目录结构

│   │   ├── Java基础/
│   │   ├── MySQL/
│   │   ├── Redis/
│   │   └── ...

## 🔄 核心工作流

| 操作 | 触发时机 | 说明 |
|------|---------|------|
| **Ingest** | 每日复盘 | DailyNotes → 提取八股文 → 更新 MOC 索引 |
| **Query** | 提问时 | 读索引 → 沿 wikilink 导航 → 返回答案 |
| **Lint** | 定期 | 检查断链、孤儿页、更新索引 |

## 🎯 知识管理原则

1. **原子化**：一个知识点一篇笔记（20 字内命名）
2. **双向链接**：`[[wikilink]]` 建立知识关联
3. **索引驱动**：MOC 页面作为导航入口
4. **AI 辅助**：Claude 自动提取、整理、维护索引

## 🚀 快速开始

1. 克隆仓库
2. 用 Obsidian 打开
3. 开始在 `DailyNotes/` 写日记
4. 让 Claude 帮你整理入库（`/整理`）
