# JVM 内存结构与运行时数据区

## 问题
JVM 内存结构是怎样的？堆和栈有什么区别？对象创建一定在堆上吗？什么是 TLAB？

## 答案

### 运行时数据区总览

JVM 运行时数据区（即内存结构）分为 5 个部分：

```
运行时数据区
├── 线程私有
│   ├── 程序计数器
│   ├── 虚拟机栈
│   └── 本地方法栈
└── 线程共享
    ├── 堆
    └── 方法区（元空间）
```

### 各区域详解

#### 1. 程序计数器（线程私有）

记录当前线程正在执行的字节码指令的地址。物理上使用**寄存器**实现。如果执行的是 native 方法，则计数器值为空（Undefined）。此区域是唯一不会发生 OOM 的内存区域。

#### 2. 虚拟机栈（线程私有）

也叫线程栈，每个线程运行时所需要的内存。一个栈由多个**栈帧**组成：
- 栈帧对应每次方法调用所占的内存
- 存储内容：**方法参数、方法内局部变量、方法返回地址**
- 一个线程同一时刻只能有一个活动栈帧，对应当前正在执行的方法
- 垃圾回收不涉及栈内存
- 方法递归过多会导致 `StackOverflowError`

#### 3. 本地方法栈（线程私有）

本地方法（native 方法）运行时所需的内存空间。native 方法不是用 Java 写的，而是用 C/C++ 编写，Java 通过本地方法接口间接调用 C/C++ 来与操作系统底层 API 交互。

#### 4. 堆（线程共享）

**保存对象实例**的核心区域。逻辑组成：

```
堆
├── 年轻代（Young Generation）
│   ├── Eden 区（伊甸园区）
│   ├── Survivor 0（S0 / From）
│   └── Survivor 1（S1 / To）
└── 老年代（Old Generation）
```

- 年轻代：存放生命周期短的对象，频繁 GC
- 老年代：存放生命周期长的对象，较少 GC
- JDK7 中堆内存在方法区的实现叫**永久代**；JDK8 改为**元空间**，移到本地内存

#### 5. 方法区（线程共享）

存储**类信息、静态变量、常量、编译后的代码、运行时常量池**。

| 版本 | 实现方式 | 位置 | 特点 |
|------|---------|------|------|
| JDK7 | 永久代（PermGen） | 堆内存 | 大小固定，容易 OOM |
| JDK8 | 元空间（Metaspace） | 本地内存 | 大小自动调整 |

#### 6. 直接内存（堆外内存）

- 不属于 JVM 内存管理，是操作系统的内存
- 常见于 NIO 操作时的数据缓冲区
- 分配和回收成本较高，但读写性能高
- Java 代码和系统代码都能访问

### 对象创建一定在堆上吗？

不一定。对象的分配流程如下：

```
新对象创建
    │
    ↓
① 逃逸分析：对象是否逃逸出方法/线程？
    │
    ├── 未逃逸 → 栈上分配（跟随方法结束自动销毁，减少 GC 压力）
    │
    └── 已逃逸
         │
         ↓
    ② TLAB 是否未满？
         │
         ├── 是 → 分配到 TLAB（线程私有，无需加锁）
         │
         └── 否 → ③ 分配到堆中（需要加锁保证线程安全）
```

**逃逸分析**：JVM 通过分析对象的动态作用域，判断对象是否可能逃逸出当前方法或线程：
- **未逃逸**：对象仅在方法内部使用
- **方法逃逸**：对象作为返回值或参数传递到其他方法
- **线程逃逸**：对象被其他线程访问（如赋值给静态变量）

如果对象未逃逸，JVM 可能将其**栈上分配**，跟随方法调用结束自动销毁，避免堆内存分配，减少 GC 压力。

### TLAB（Thread Local Allocation Buffer）

**问题引入**：多线程同时 new 对象时，JVM 为防止内存重复分配需要加互斥锁，降低分配速度。

**解决方案**：TLAB 是每个线程在 Eden 区专属的一块区域。当线程的 TLAB 未满时，可以直接分配内存，**不需要加锁**，提高内存分配速度。

**关键点**：
- 当一个线程创建时，JVM 会为其分配一个固定大小的 TLAB 空间
- TLAB 仅优化内存分配过程的线程安全，与对象本身的线程安全性无关

### 为什么区分堆和栈？

1. **分开管理**：堆由垃圾回收器管理，栈由编译器和虚拟机执行管理
2. **互不影响**：堆溢出不影响栈，栈溢出不影响堆
3. **数据隔离**：线程独享的局部变量放栈上做隔离，共享数据放堆上统一管理
4. **提升性能**：栈上分配效率很高，堆上分配和回收相对复杂，各自优化提升整体效率
