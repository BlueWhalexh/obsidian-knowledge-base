# 双亲委派与类加载器

## 问题
什么是类加载器？有哪些类加载器？什么是双亲委派？为什么要用双亲委派？如何破坏双亲委派？

## 答案

### 类加载器概述

类加载器的作用是将**字节码文件（.class）**加载到 JVM 内存中。

### 4 层类加载器

| 类加载器 | 加载范围 | 说明 |
|---------|---------|------|
| **Bootstrap ClassLoader（启动类加载器）** | `JAVA_HOME/jre/lib` 下的类 | 由 C/C++ 实现，无法直接访问 |
| **Extension ClassLoader（扩展类加载器）** | `JAVA_HOME/jre/lib/ext` 下的类 | 上级为 Bootstrap |
| **Application ClassLoader（应用程序类加载器）** | classpath 目录下的类（项目代码目录） | 上级为 Extension |
| **自定义类加载器** | 自定义加载规则 | 上级为 Application，可绕过/破坏双亲委派 |

注意：类加载器的父子关系不是以继承方式实现，而是**组合**关系。

### 双亲委派流程

双亲委派模型规定了加载类时类加载器的行为：

```
收到类加载请求
    │
    ↓
① 先检查类是否已被加载
    │
    ├── 已加载 → 直接返回
    │
    └── 未加载
         │
         ↓
② 委派给父类加载器加载
    │
    ├── 父加载器找到类 → 加载并返回
    │
    └── 父加载器未找到（ClassNotFoundException）
         │
         ↓
③ 自己调用 findClass() 加载
```

**工作过程**：如果一个类加载器收到了类加载请求，它首先不会自己尝试加载，而是把请求委派给父类加载器。每一层都是如此，最终传送到启动类加载器。只有当父加载器反馈无法完成加载时，子加载器才会尝试自己加载。

### 双亲委派的好处

**1. 保证类的唯一性**
通过双亲委派机制避免同一个类被重复加载。当父加载器已经加载后，无需重复加载。

**2. 保证类的安全性**
防止用户自定义类覆盖核心类。例如用户编写一个 `java.lang.String` 类，由于双亲委派，启动类加载器会优先加载 JDK 自带的 String 类，用户自定义的不会被加载。

### 破坏双亲委派

双亲委派的实现代码集中在 `java.lang.ClassLoader` 的 `loadClass()` 方法中：

```java
// loadClass() 的逻辑：
// 1. 检查类是否已被加载
// 2. 若未加载，调用父加载器的 loadClass()
// 3. 若父加载器为空，使用启动类加载器
// 4. 若父类加载失败（ClassNotFoundException），调用自己的 findClass()
```

**破坏方式**：自定义类加载器，重写 `loadClass()` 方法，内部逻辑不遵循双亲委派。

### 经典案例：Tomcat 破坏双亲委派

**原因**：一个 Tomcat 可以同时运行多个 Web 应用，不同应用可能依赖相同类库但版本不同，这些类库的全路径名相同。双亲委派机制无法重复加载同一个类。

**解决方案**：Tomcat 为每个应用提供独立的 **WebappClassLoader**（自定义类加载器），内部逻辑是：针对私有类**不遵循双亲委派**，而是自己加载。即使全类名相同，由于类加载器不同，JVM 也会视为不同类。

**优化**：对于公共的 jar 包（如 Spring），各应用版本相同时，Tomcat 提供 **SharedClassLoader** 统一加载，避免重复加载，节省内存。

### 类的生命周期

类的完整生命周期：**加载 → 链接（验证 → 准备 → 解析）→ 初始化 → 使用 → 卸载**

| 阶段 | 核心任务 | 关键规则 |
|------|---------|---------|
| **加载** | 读取字节码，生成 Class 对象 | 类加载器负责，支持自定义逻辑 |
| **验证** | 确保字节码符合 JVM 规范 | 文件格式、语义、符号引用验证 |
| **准备** | 为类变量分配内存并设置零值 | `static int value = 42` 此阶段初始化为 0；`static final` 常量直接赋值 |
| **解析** | 符号引用替换为直接引用 | 可能延迟到初始化后 |
| **初始化** | 执行 `<clinit>()` 方法，完成静态赋值和静态代码块 | 父类优先，线程安全，被动引用不触发 |
| **卸载** | 回收类元数据 | 需同时满足：实例已回收、Class 对象无引用、ClassLoader 已回收 |
