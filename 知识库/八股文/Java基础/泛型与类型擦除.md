# 泛型与类型擦除

## 问题
泛型是什么？有什么好处？什么是类型擦除？为什么需要类型擦除？泛型通配符怎么用？

## 答案

### 泛型是什么

泛型（Generics）是 Java 5 引入的特性，允许在定义类、接口和方法时使用**类型参数**（Type Parameter）。泛型提供了一种**参数化类型**的能力，使得代码可以适用于多种数据类型，同时保证类型安全。

```java
// 没有泛型：需要强制转换，可能 ClassCastException
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0); // 需要强转
list.add(123); // 编译不报错，运行时可能出错

// 有泛型：类型安全，无需强转
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0); // 无需强转
// list.add(123); // 编译报错！类型安全
```

### 泛型的好处

#### 1. 类型安全（编译时检查）

泛型在**编译时**进行类型检查，将运行时的 `ClassCastException` 提前到编译时发现。

```java
// 没有泛型：运行时才发现错误
List list = new ArrayList();
list.add("hello");
list.add(123); // 编译通过
for (Object obj : list) {
    String s = (String) obj; // 运行时 ClassCastException！
}

// 有泛型：编译时就发现错误
List<String> list = new ArrayList<>();
list.add("hello");
// list.add(123); // 编译错误！类型不匹配
for (String s : list) {
    System.out.println(s); // 无需强转，安全
}
```

#### 2. 代码复用（消除重复代码）

泛型使得一段代码可以适用于多种类型，避免为每种类型编写重复的代码。

```java
// 没有泛型：需要为每种类型写一个类
public class IntegerBox {
    private Integer value;
    public void set(Integer value) { this.value = value; }
    public Integer get() { return value; }
}
public class StringBox {
    private String value;
    public void set(String value) { this.value = value; }
    public String get() { return value; }
}

// 有泛型：一个类适用于所有类型
public class Box<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}

Box<Integer> intBox = new Box<>();
intBox.set(123);

Box<String> strBox = new Box<>();
strBox.set("hello");
```

#### 3. 消除强制类型转换

```java
// 没有泛型
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0); // 需要强转

// 有泛型
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0); // 无需强转，编译器自动处理
```

### 类型擦除是什么

**类型擦除（Type Erasure）** 是 Java 泛型的实现机制：泛型信息**只在编译时存在**，编译后会被**擦除**，替换为原始类型（Raw Type）。

**擦除规则**：
- `<T>` 无界类型参数 → 擦除为 `Object`
- `<T extends Comparable>` 有界类型参数 → 擦除为上界 `Comparable`

```java
// 编译前（带泛型）
public class Box<T> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}

// 编译后（擦除泛型）
public class Box {
    private Object value;
    public void set(Object value) { this.value = value; }
    public Object get() { return value; }
}
```

**有界泛型的擦除**：
```java
// 编译前
public class Box<T extends Number> {
    private T value;
    public void set(T value) { this.value = value; }
    public T get() { return value; }
}

// 编译后（擦除为上界 Number）
public class Box {
    private Number value;
    public void set(Number value) { this.value = value; }
    public Number get() { return value; }
}
```

**证明类型擦除**：
```java
List<String> stringList = new ArrayList<>();
List<Integer> intList = new ArrayList<>();

// 编译后两者是同一个类
System.out.println(stringList.getClass() == intList.getClass()); // true

// 通过反射可以绕过泛型检查（因为运行时没有泛型信息）
List<Integer> list = new ArrayList<>();
list.getClass().getMethod("add", Object.class).invoke(list, "abc");
System.out.println(list.get(0)); // "abc"（运行时没有泛型检查）
```

**类型擦除的插入类型转换**：
```java
// 编译前
List<String> list = new ArrayList<>();
list.add("hello");
String s = list.get(0);

// 编译后（编译器自动插入强转）
List list = new ArrayList();
list.add("hello");
String s = (String) list.get(0); // 编译器自动插入的强转
```

**类型擦除的桥方法（Bridge Method）**：
```java
// 编译前
public class IntegerBox extends Box<Integer> {
    @Override
    public void set(Integer value) { ... }
}

// 编译后（编译器生成桥方法保持多态性）
public class IntegerBox extends Box {
    @Override
    public void set(Integer value) { ... }
    
    // 编译器生成的桥方法
    public void set(Object value) {
        set((Integer) value); // 强转后调用实际方法
    }
}
```

### 为什么需要类型擦除

**核心原因：向后兼容（Backward Compatibility）**

1. **兼容旧版本 JVM**：Java 5 引入泛型时，已经存在大量运行在旧版 JVM 上的 Java 程序。如果泛型需要 JVM 支持，旧版 JVM 就无法运行新代码
2. **兼容旧代码**：泛型代码和非泛型代码可以互操作

```java
// Java 1.4 的旧代码
List list = new ArrayList();
list.add("hello");

// Java 5+ 的新代码可以使用旧代码
List<String> stringList = list; // 可以兼容（有 unchecked 警告）
```

3. **不需要修改 JVM 指令集**：类型擦除在编译器层面完成，JVM 无需知道泛型的存在

**类型擦除带来的限制**：

| 限制 | 原因 | 示例 |
|------|------|------|
| 不能用基本类型作为类型参数 | 擦除后为 Object，基本类型不是 Object | `List<int>` 不行，要用 `List<Integer>` |
| 不能创建泛型数组 | 数组在运行时有类型检查 | `new T[]` 不行 |
| 不能用 instanceof 判断泛型类型 | 运行时没有泛型信息 | `obj instanceof List<String>` 不行 |
| 不能创建泛型类的实例 | 擦除后不知道具体类型 | `new T()` 不行 |
| 不能在静态上下文中使用类型参数 | 靪态成员属于类，不绑定具体类型 | `static T value` 不行 |

### 泛型通配符

泛型通配符用于在泛型类型不确定时提供灵活性。主要有三种形式：

#### 1. 无界通配符 `<?>`

表示**任意类型**，通常用于读取操作。

```java
// 可以接受任何类型的 List
public void printList(List<?> list) {
    for (Object item : list) {
        System.out.println(item);
    }
}

printList(Arrays.asList("a", "b", "c")); // List<String>
printList(Arrays.asList(1, 2, 3));       // List<Integer>
```

**使用场景**：
- 只关心集合中的元素类型是否是 Object
- 方法不依赖于具体的类型参数

#### 2. 上界通配符 `<? extends T>`（协变）

表示**T 或 T 的子类**，用于**读取**数据（生产者 Producer）。

```java
// 只能读取 Number 或 Number 的子类
public double sum(List<? extends Number> list) {
    double sum = 0;
    for (Number num : list) { // 可以安全地读取为 Number
        sum += num.doubleValue();
    }
    return sum;
}

sum(Arrays.asList(1, 2, 3));       // List<Integer>（Integer extends Number）
sum(Arrays.asList(1.1, 2.2, 3.3)); // List<Double>（Double extends Number）
```

**PECS 原则：Producer Extends** — 如果你要**从集合中读取**数据，使用 `extends`。

**限制**：不能往 `List<? extends Number>` 中添加元素（除了 null），因为不知道具体类型：
```java
List<? extends Number> list = new ArrayList<Integer>();
list.add(1);    // 编译错误！不知道是 Integer 还是 Double
list.add(null); // 可以（null 是所有类型的子类）
```

#### 3. 下界通配符 `<? super T>`（逆变）

表示**T 或 T 的父类**，用于**写入**数据（消费者 Consumer）。

```java
// 只能添加 Integer 或 Integer 的子类
public void addNumbers(List<? super Integer> list) {
    list.add(1);
    list.add(2);
    list.add(3);
}

List<Number> numberList = new ArrayList<>();
addNumbers(numberList); // 可以，Number 是 Integer 的父类

List<Object> objectList = new ArrayList<>();
addNumbers(objectList); // 可以，Object 是 Integer 的父类
```

**PECS 原则：Consumer Super** — 如果你要**往集合中写入**数据，使用 `super`。

**限制**：从 `List<? super Integer>` 中读取只能得到 Object：
```java
List<? super Integer> list = new ArrayList<Number>();
list.add(1);         // 可以
Object obj = list.get(0); // 只能读为 Object，不能读为 Integer
Integer num = list.get(0); // 编译错误！
```

#### PECS 原则总结

**PECS = Producer Extends, Consumer Super**

| 场景 | 通配符 | 说明 |
|------|--------|------|
| 只读（生产者） | `<? extends T>` | 从集合中读取 T 类型的数据 |
| 只写（消费者） | `<? super T>` | 往集合中写入 T 类型的数据 |
| 读写都可以 | 确定的类型参数 `<T>` | 既读又写时，使用确切类型 |

**经典示例 — Collections.copy()**：
```java
// src 是生产者（只读），用 extends
// dest 是消费者（只写），用 super
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (int i = 0; i < src.size(); i++) {
        dest.set(i, src.get(i));
    }
}
```

### 泛型的使用场景

**1. 泛型类**：
```java
public class Result<T> {
    private int code;
    private String message;
    private T data;
    
    public static <T> Result<T> success(T data) {
        Result<T> result = new Result<>();
        result.setCode(200);
        result.setData(data);
        return result;
    }
}

Result<User> result = Result.success(user);
Result<List<Order>> result = Result.success(orders);
```

**2. 泛型方法**：
```java
public static <T extends Comparable<T>> T max(T a, T b) {
    return a.compareTo(b) >= 0 ? a : b;
}

String maxStr = max("abc", "def"); // "def"
int maxInt = max(1, 2);            // 2
```

**3. 泛型接口**：
```java
public interface Repository<T, ID> {
    T findById(ID id);
    List<T> findAll();
    void save(T entity);
    void delete(ID id);
}

public class UserRepository implements Repository<User, Long> {
    @Override
    public User findById(Long id) { ... }
    // ...
}
```
