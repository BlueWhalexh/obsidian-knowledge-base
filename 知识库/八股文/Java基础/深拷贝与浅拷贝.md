# 深拷贝与浅拷贝

## 问题
什么是深拷贝和浅拷贝？它们之间有什么区别？

## 答案

### 核心概念

深拷贝（Deep Copy）和浅拷贝（Shallow Copy）是复制对象或数据结构时使用的两种不同方法，区别主要在于**拷贝的程度**和**对原始对象内部结构的影响**。

### 浅拷贝

浅拷贝创建一个新对象，但只复制原始对象的**基本数据类型字段**或**引用地址**，不复制引用指向的对象。

**特点**：
- 基本数据类型字段：值被复制到新对象中，互不影响
- 引用类型字段：新对象和原始对象**共享同一个引用对象**
- 修改新对象的引用类型字段会影响原始对象

**实现方式**：
- 实现 `Cloneable` 接口，重写 `Object.clone()` 方法（默认就是浅拷贝）

### 深拷贝

深拷贝创建一个新对象，并**递归复制**原始对象的所有字段和引用指向的对象，不仅仅是复制引用本身。

**特点**：
- 新对象和原始对象完全独立
- 修改新对象的任何字段都**不会影响**原始对象
- 拷贝整个对象图，包括对象内部嵌套的所有对象

**实现方式**：
1. 重写 `clone()` 方法，手动对引用类型字段也调用 `clone()`
2. 通过**序列化/反序列化**实现（对象需实现 `Serializable` 接口）
3. 使用 JSON 转换工具（如 Jackson、Gson）

### User/Address 示例

```java
class User implements Cloneable {
    private String name;
    private String password;
    private Address address;

    // 浅拷贝
    @Override
    protected User clone() throws CloneNotSupportedException {
        return (User) super.clone();
    }

    // 深拷贝：手动克隆引用对象
    protected User deepClone() throws CloneNotSupportedException {
        User user = (User) super.clone();
        user.address = this.address.clone(); // 递归复制 Address
        return user;
    }
}

class Address implements Cloneable {
    private String province;
    private String city;
    private String area;

    @Override
    protected Address clone() throws CloneNotSupportedException {
        return (Address) super.clone();
    }
}
```

### 浅拷贝现象

```
user1 = new User("张三", "123", new Address("广东", "深圳", "南山"))
user2 = user1.clone()  // 浅拷贝

user1.address 和 user2.address 指向同一个 Address 对象
修改 user1.address 的城市为 "广州"，user2.address 的城市也会变为 "广州"
```

### 深拷贝现象

```
user1 = new User("张三", "123", new Address("广东", "深圳", "南山"))
user2 = user1.deepClone()  // 深拷贝

user1.address 和 user2.address 是两个独立的 Address 对象
修改 user1.address 的城市为 "广州"，user2.address 的城市仍为 "深圳"
```

### 对比总结

| 对比项 | 浅拷贝 | 深拷贝 |
|---|---|---|
| 基本类型字段 | 复制值，互不影响 | 复制值，互不影响 |
| 引用类型字段 | 共享引用，互相影响 | 递归复制，互不影响 |
| 对象独立性 | 不完全独立 | 完全独立 |
| 实现复杂度 | 简单（直接 clone） | 较复杂（需递归克隆） |
| 性能 | 较快 | 较慢（需复制整个对象图） |
| 内存占用 | 较少 | 较多 |

### 数组的特殊情况

**数组的 `clone()` 和 `Arrays.copyOf()` 是浅拷贝还是深拷贝？**

分两种情况：

| 数组类型 | `clone()` / `copyOf()` | 原因 |
|---------|----------------------|------|
| 基本类型数组 `int[]`, `double[]` | ✅ 深拷贝 | 元素是值，直接复制 |
| 引用类型数组 `String[]`, `User[]` | ⚠️ 浅拷贝 | 只复制引用地址，原数组和新数组指向同一批对象 |

**常用方案**：

```java
// 基本类型数组 — copyOf 即可实现深拷贝
int[] original = {1, 2, 3};
int[] copy = Arrays.copyOf(original, original.length);
copy[0] = 999;
// original[0] 仍然是 1 ✅

// 二维数组 — copyOf 只拷贝第一层（浅拷贝）
int[][] matrix = {{1, 2}, {3, 4}};
int[][] copy = Arrays.copyOf(matrix, matrix.length);
copy[0][0] = 999;
// matrix[0][0] 也变成了 999 ❌ 因为内层数组是引用！
```

> ⚠️ **实战坑**：LeetCode 等刷题平台做原地修改算法时，Java 数组参数是引用传递。如果你在函数内修改了输入数组，测试框架拿到的也是被修改后的版本。对比测试结果时需要先 `Arrays.copyOf()` 保存原始数组。

### Quick Reference

| 场景 | 推荐方式 |
|------|---------|
| 基本类型一维数组 | `Arrays.copyOf(original, len)` |
| 引用类型一维数组 | 遍历 + 逐个 clone 元素，或用序列化 |
| 多维数组 | 逐层 copyOf / 序列化 |
| 对象 | 重写 `clone()` 递归克隆引用字段 |
