# 迭代器的实现原理

## 问题
迭代器是什么？为什么需要迭代器？ArrayList/LinkedList/HashMap 的迭代器实现有什么区别？增强 for 循环的底层原理是什么？

## 答案

### 迭代器是什么

迭代器（Iterator）是 Java 集合框架中用于**遍历集合元素**的接口，定义在 `java.util` 包中。它提供了一种**统一的方式来访问不同集合中的元素**，而无需了解集合的内部实现细节。

```java
public interface Iterator<E> {
    boolean hasNext(); // 是否还有下一个元素
    E next();          // 返回下一个元素
    default void remove() { ... } // 删除上一次 next() 返回的元素
    default void forEachRemaining(Consumer<? super E> action) { ... }
}
```

### 为什么需要迭代器（迭代器模式）

**核心问题**：不同的集合底层数据结构不同（数组、链表、红黑树），遍历方式也不同。如果没有迭代器，使用者需要了解每种集合的内部结构才能遍历。

**迭代器模式（Iterator Pattern）** 是一种**行为型设计模式**，核心思想是：
- 将**遍历行为**从集合中**分离出来**，封装到迭代器对象中
- 集合只负责存储数据，迭代器负责遍历数据
- 对外提供统一的遍历接口，隐藏内部实现

**迭代器的优势**：

| 优势 | 说明 |
|------|------|
| **统一接口** | 所有集合都用 hasNext()/next() 遍历，无需关心底层结构 |
| **解耦** | 遍历逻辑与集合存储逻辑分离，符合单一职责原则 |
| **多种遍历** | 同一集合可以提供不同的迭代器（如正序、倒序） |
| **并发安全** | 可以在迭代器中实现并发修改检测（fail-fast） |
| **延迟加载** | 迭代器可以实现惰性求值，按需获取元素 |

**生活比喻**：迭代器就像**图书馆的索引卡片**。无论图书是按字母排序（TreeSet）、按编号摆放（ArrayList）、还是分散在不同书架（HashMap），索引卡片都提供统一的查找方式。

### Iterator 接口方法详解

```java
public interface Iterator<E> {
    
    // 判断是否还有下一个元素
    boolean hasNext();
    
    // 返回下一个元素，并将指针后移
    E next();
    
    // 删除上一次 next() 返回的元素（可选操作）
    default void remove() {
        throw new UnsupportedOperationException("remove");
    }
    
    // 对剩余元素执行操作（Java 8 新增）
    default void forEachRemaining(Consumer<? super E> action) {
        Objects.requireNonNull(action);
        while (hasNext())
            action.accept(next());
    }
}
```

**使用示例**：
```java
List<String> list = Arrays.asList("A", "B", "C");
Iterator<String> it = list.iterator();

while (it.hasNext()) {
    String element = it.next();
    System.out.println(element);
    // 安全删除：使用 iterator.remove() 而非 list.remove()
    if ("B".equals(element)) {
        it.remove(); // 安全删除，不会抛 ConcurrentModificationException
    }
}
```

**Iterator vs Iterable**：
- `Iterable`：表示该集合**可被迭代**，提供 `iterator()` 方法返回迭代器
- `Iterator`：实际的**迭代器对象**，负责遍历逻辑
- `for-each` 语法要求集合实现 `Iterable` 接口

```java
public interface Iterable<T> {
    Iterator<T> iterator(); // 返回迭代器
}
```

### ArrayList 的迭代器实现

ArrayList 的迭代器是内部类 `Itr`，基于**数组下标**实现遍历。

```java
private class Itr implements Iterator<E> {
    int cursor;       // 下一个要返回的元素的下标
    int lastRet = -1; // 上一次返回的元素的下标（-1表示没有）
    int expectedModCount = modCount; // 期望的修改次数（用于fail-fast）

    public boolean hasNext() {
        return cursor != size; // cursor 不等于元素数量，说明还有元素
    }

    @SuppressWarnings("unchecked")
    public E next() {
        checkForComodification(); // 检查是否被并发修改
        int i = cursor;
        if (i >= size)
            throw new NoSuchElementException();
        Object[] elementData = ArrayList.this.elementData;
        if (i >= elementData.length)
            throw new ConcurrentModificationException();
        cursor = i + 1; // cursor 后移
        return (E) elementData[lastRet = i]; // 返回元素，记录 lastRet
    }

    public void remove() {
        if (lastRet < 0)
            throw new IllegalStateException();
        checkForComodification();
        try {
            ArrayList.this.remove(lastRet); // 调用 ArrayList 的 remove
            cursor = lastRet; // cursor 回退（因为元素前移了）
            lastRet = -1;
            expectedModCount = modCount; // 更新期望修改次数
        } catch (IndexOutOfBoundsException ex) {
            throw new ConcurrentModificationException();
        }
    }

    final void checkForComodification() {
        if (modCount != expectedModCount)
            throw new ConcurrentModificationException(); // fail-fast
    }
}
```

**ArrayList 迭代器的特点**：
- 基于数组下标遍历，`next()` 直接通过 `elementData[cursor]` 访问，时间复杂度 O(1)
- **fail-fast 机制**：通过 `modCount` 检测并发修改，一旦检测到立即抛出 `ConcurrentModificationException`
- `remove()` 操作会调用 `ArrayList.remove(lastRet)`，需要移动数组元素，时间复杂度 O(n)

**fail-fast 机制详解**：
```java
// ArrayList 维护一个修改计数器
protected transient int modCount = 0;

// 每次 add/remove 都会增加 modCount
public boolean add(E e) {
    modCount++; // 修改次数+1
    // ...
}

// 创建迭代器时记录当前 modCount
Iterator<String> it = list.iterator(); // expectedModCount = modCount

// 遍历时检查 modCount 是否变化
String s = it.next(); // 检查 modCount == expectedModCount

// 如果在遍历期间直接调用 list.add() 或 list.remove()
list.add("X"); // modCount++，不等于 expectedModCount
it.next();      // 抛出 ConcurrentModificationException！
```

### LinkedList 的迭代器实现

LinkedList 的迭代器是内部类 `ListItr`，基于**双向链表的节点引用**实现遍历。

```java
private class ListItr implements ListIterator<E> {
    private Node<E> lastReturned; // 上一次返回的节点
    private Node<E> next;         // 下一个要返回的节点
    private int nextIndex;        // 下一个节点的下标
    private int expectedModCount = modCount;

    ListItr(int index) {
        // 根据 index 定位到起始节点
        next = (index == size) ? null : node(index);
        nextIndex = index;
    }

    public boolean hasNext() {
        return nextIndex < size;
    }

    public E next() {
        checkForComodification();
        if (!hasNext()) throw new NoSuchElementException();
        lastReturned = next;
        next = next.next; // 沿着链表的 next 指针移动
        nextIndex++;
        return lastReturned.item;
    }

    // LinkedList 还支持向前遍历（ListIterator 特有）
    public boolean hasPrevious() {
        return nextIndex > 0;
    }

    public E previous() {
        checkForComodification();
        if (!hasPrevious()) throw new NoSuchElementException();
        lastReturned = next = (next == null) ? last : next.prev;
        nextIndex--;
        return lastReturned.item;
    }
}
```

**LinkedList 迭代器的特点**：
- 基于节点引用遍历，`next()` 通过 `node.next` 移动，时间复杂度 O(1)
- 支持**双向遍历**（`hasPrevious()` / `previous()`），因为 LinkedList 是双向链表
- `remove()` 操作直接修改节点指针，时间复杂度 O(1)（不需要像 ArrayList 那样移动元素）
- 同样有 fail-fast 机制

### HashMap 的迭代器实现

HashMap 的迭代器是抽象内部类 `HashIterator`，三个具体迭代器（KeySet/Values/EntrySet）都继承自它。

```java
abstract class HashIterator {
    Node<K,V> next;        // 下一个要返回的节点
    Node<K,V> current;     // 当前节点
    int expectedModCount;  // 期望的修改次数
    int index;             // 当前遍历到的桶的下标

    HashIterator() {
        expectedModCount = modCount;
        Node<K,V>[] t = table;
        current = next = null;
        index = 0;
        // 找到第一个非空桶
        if (t != null && size > 0) {
            do {} while (index < t.length && (next = t[index++]) == null);
        }
    }

    public final boolean hasNext() {
        return next != null;
    }

    final Node<K,V> nextNode() {
        Node<K,V>[] t;
        Node<K,V> e = next;
        checkForComodification();
        if (e == null)
            throw new NoSuchElementException();
        // 如果当前桶的链表/红黑树遍历完了，找下一个非空桶
        if ((next = (current = e).next) == null && (t = table) != null) {
            do {} while (index < t.length && (next = t[index++]) == null);
        }
        return e;
    }

    public final void remove() {
        Node<K,V> p = current;
        if (p == null)
            throw new IllegalStateException();
        checkForComodification();
        current = null;
        K key = p.key;
        removeNode(hash(key), key, null, false, false);
        expectedModCount = modCount;
    }
}

// 三种具体迭代器
final class KeyIterator extends HashIterator implements Iterator<K> {
    public final K next() { return nextNode().key; }
}

final class ValueIterator extends HashIterator implements Iterator<V> {
    public final V next() { return nextNode().value; }
}

final class EntryIterator extends HashIterator implements Iterator<Map.Entry<K,V>> {
    public final Map.Entry<K,V> next() { return nextNode(); }
}
```

**HashMap 迭代器的特点**：
- 遍历顺序：先遍历数组（桶），再遍历每个桶中的链表/红黑树
- **无序遍历**：元素的遍历顺序与插入顺序无关（取决于 hash 值和桶的位置）
- `nextNode()` 中的 `do {} while (index < t.length && (next = t[index++]) == null)` 用于跳过空桶
- 如果在遍历过程中发生扩容，迭代器会遍历到新数组（因为 `table` 引用会更新）

**遍历 HashMap 的三种方式**：
```java
Map<String, Integer> map = new HashMap<>();
map.put("a", 1);
map.put("b", 2);
map.put("c", 3);

// 方式1：遍历 key
for (String key : map.keySet()) {
    System.out.println(key);
}

// 方式2：遍历 value
for (Integer value : map.values()) {
    System.out.println(value);
}

// 方式3：遍历 entry（推荐，同时获取 key 和 value）
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}

// 方式4：使用 forEach（Java 8+）
map.forEach((key, value) -> System.out.println(key + " = " + value));
```

### 增强 for 循环底层也是迭代器

**增强 for 循环（for-each）** 是 Java 5 引入的语法糖，底层**完全依赖迭代器**实现。

```java
// 增强 for 循环
List<String> list = Arrays.asList("A", "B", "C");
for (String s : list) {
    System.out.println(s);
}

// 编译器转换后的等价代码（反编译可见）
List<String> list = Arrays.asList("A", "B", "C");
Iterator<String> iterator = list.iterator();
while (iterator.hasNext()) {
    String s = iterator.next();
    System.out.println(s);
}
```

**证明 for-each 底层是迭代器**：
```java
// 在 for-each 中调用 list.remove() 会抛出 ConcurrentModificationException
for (String s : list) {
    if ("B".equals(s)) {
        list.remove(s); // 抛出 ConcurrentModificationException！
        // 因为底层是迭代器在遍历，外部修改了 modCount
    }
}
```

**for-each 的要求**：
- 集合必须实现 `Iterable` 接口（Collection 接口继承了 Iterable）
- 数组也可以用 for-each，但底层不是迭代器，而是普通的 for 循环

```java
// 数组的 for-each 编译后
int[] arr = {1, 2, 3};
for (int n : arr) {
    System.out.println(n);
}
// 等价于：
for (int i = 0; i < arr.length; i++) {
    int n = arr[i];
    System.out.println(n);
}
```

### 迭代器使用注意事项

**1. 遍历时删除元素**：
```java
// 错误方式：ConcurrentModificationException
for (String s : list) {
    if ("B".equals(s)) {
        list.remove(s); // 报错！
    }
}

// 正确方式1：使用 Iterator.remove()
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if ("B".equals(it.next())) {
        it.remove(); // 安全删除
    }
}

// 正确方式2：使用 removeIf（Java 8+）
list.removeIf(s -> "B".equals(s));
```

**2. fail-fast vs fail-safe**：

| 特性 | fail-fast | fail-safe |
|------|-----------|-----------|
| 代表 | ArrayList, HashMap, HashSet | CopyOnWriteArrayList, ConcurrentHashMap |
| 行为 | 检测到并发修改立即抛异常 | 不抛异常，遍历的是快照 |
| 原理 | 维护 modCount 计数器 | 遍历的是集合的副本 |
| 性能 | 高 | 较低（需要复制） |
