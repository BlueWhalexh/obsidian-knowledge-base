# Spring Boot 自定义线程池

## 问题
Spring Boot 中为什么要自定义线程池？需要关注哪些监控指标？

## 答案

**为什么要自定义**：
1. **等待队列**：默认是无界队列 → 容易 OOM
2. **拒绝策略**：可根据业务自定义（如记录日志、降级处理）
3. **监控指标**：队列大小、活跃线程数、CPU 利用率

**核心监控指标**：
- 等待队列数量
- 空闲线程数量
- CPU 利用率
