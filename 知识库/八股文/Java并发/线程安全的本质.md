# 线程安全的本质

## 问题
什么是线程安全？什么是 JMM？原子性、可见性、有序性分别是什么？如何保证线程安全？多线程交替打印问题？

## 答案

### JMM 是什么

**JMM（Java Memory Model，Java 内存模型）** 是 Java 虚拟机规范中定义的一套**抽象规范**，用于屏蔽不同硬件和操作系统的内存访问差异，定义了多线程程序中变量的访问规则。

**JMM 的核心结构**：

```
线程A的工作内存 ←→ 主内存 ←→ 线程B的工作内存
     ↑                              ↑
  本地变量副本                    本地变量副本
```

- **主内存（Main Memory）**：所有线程共享的内存区域，存储共享变量的**主副本**
- **工作内存（Working Memory）**：每个线程私有的内存区域，存储共享变量的**本地副本**

**JMM 的核心规则**：
1. 所有共享变量都存储在**主内存**中
2. 每个线程有自己的**工作内存**，保存了共享变量的副本
3. 线程对变量的操作必须在**工作内存**中进行，不能直接读写主内存
4. 线程间变量的传递必须通过**主内存**中转

**JMM 带来的问题**：
- 线程 A 修改了变量，先写入工作内存，还未同步到主内存
- 线程 B 从主内存读取到的是**旧值**
- 这就是**可见性问题**的根源

**JMM 与 JVM 内存结构的区别**：
- JMM 是**抽象规范**，定义的是多线程环境下的内存访问规则
- JVM 内存结构是**实际实现**，定义的是 JVM 运行时的内存划分（堆、栈、方法区等）
- JMM 中的主内存对应 JVM 堆中的实例数据，工作内存对应 JVM 栈中的局部变量

### 线程安全的三个核心要素

线程安全的本质是：在多线程并发访问共享资源时，程序的执行结果始终符合预期，不会出现数据错乱或逻辑错误。根源在于以下三个层面的问题：

#### 1. 原子性（Atomicity）

**定义**：一个操作或一系列操作要么**全部执行成功**，要么**全部不执行**，不会被其他线程中断。中间状态对其他线程不可见。

**生活比喻**：银行转账是原子性操作——A 账户扣款和 B 账户入账必须同时成功或同时失败，不能只完成一半。

**Java 中哪些操作是原子的？**
- 基本类型的赋值（`int a = 1`）是原子的
- 引用类型的赋值（`Object obj = new Object()`）是原子的
- `volatile` 修饰的 long/double 的读写是原子的（JMM 保证）

**Java 中哪些操作不是原子的？**
- `i++`（读取-修改-写入，三步操作）
- `i = i + 1`
- 复合操作（先检查再执行，如 `if (map.get(key) == null) map.put(key, value)`）

**i++ 字节码分析（4条指令）**：

```java
// i++ 对应以下字节码步骤：
getstatic i    // 1. 从主存读取静态变量 i 到操作数栈
iconst_1       // 2. 将常量 1 压入操作数栈
iadd           // 3. 栈顶两个值相加（i+1）
putstatic i    // 4. 将结果写回静态变量 i
```

多线程情况下，每个线程分别执行自己的四条字节码指令，且多线程是**抢占式**获得 CPU 执行权。线程切换可能发生在任意字节码指令之间，导致操作被"切割"，即多个线程的多条字节码指令存在交叉执行。

**假设线程 A 和线程 B 同时执行 `i++`（初始值 i=0）**：
1. 线程 A 读取 i=0，线程 B 也读取 i=0
2. 线程 A 计算 i+1=1，线程 B 计算 i+1=1
3. 线程 A 写回 i=1，线程 B 写回 i=1
4. **最终结果**：i=1（期望值为 2），出现错误。

**JMM 如何保证原子性？**
- `synchronized`：互斥锁，保证同一时刻只有一个线程执行临界区代码
- `Lock`（ReentrantLock）：显式锁，功能更丰富
- 原子类（AtomicInteger 等）：基于 CAS（Compare-And-Swap）无锁编程

#### 2. 可见性（Visibility）

**定义**：当一个线程修改了共享变量的值，其他线程能够**立即看到**修改后的值。

**为什么会有可见性问题？**
- 每个线程有自己的工作内存（CPU 缓存的抽象）
- 线程修改变量时，先写入工作内存，再同步到主内存
- 其他线程读取变量时，从自己的工作内存读取（可能是旧值）

```java
// 可见性问题示例
public class VisibilityDemo {
    private boolean flag = true; // 没有 volatile

    public void writer() {
        flag = false; // 线程A修改
    }

    public void reader() {
        while (flag) { // 线程B可能永远读到 true（从工作内存读取）
            // 死循环
        }
    }
}
```

**JMM 如何保证可见性？**
- `volatile`：强制每次读写都直接操作主内存，禁止缓存优化
- `synchronized`：解锁前将工作内存中的变量刷新到主内存，加锁时从主内存读取最新值
- `final`：final 字段在构造函数中初始化后，对所有线程可见

```java
// volatile 保证可见性
public class VisibilityDemo {
    private volatile boolean flag = true; // 加 volatile

    public void writer() {
        flag = false; // 立即刷新到主内存
    }

    public void reader() {
        while (flag) { // 每次从主内存读取最新值
            // 能正常退出
        }
    }
}
```

#### 3. 有序性（Ordering）

**定义**：程序代码的执行顺序与代码的书写顺序一致。

**为什么会有有序性问题？**

编译器和 CPU 为了提高性能，会对指令进行**重排序**（Reordering）：

1. **编译器重排序**：编译器在不改变单线程程序语义的前提下，调整指令顺序以优化性能
2. **指令级并行重排序**：CPU 采用流水线、乱序执行等技术，将多条指令重叠执行
3. **内存系统重排序**：由于缓存和读写缓冲区的存在，加载和存储操作可能乱序

**经典案例：双重检查锁定（DCL）单例**

```java
public class Singleton {
    private static Singleton instance; // 没有 volatile

    public static Singleton getInstance() {
        if (instance == null) {           // 第一次检查
            synchronized (Singleton.class) {
                if (instance == null) {    // 第二次检查
                    instance = new Singleton(); // 问题在这里！
                }
            }
        }
        return instance;
    }
}
```

`new Singleton()` 实际上分三步：
1. 分配内存空间
2. 初始化对象
3. 将引用指向内存地址

由于重排序，步骤 2 和 3 可能被交换：
1. 分配内存空间
2. 将引用指向内存地址（此时对象还未初始化！）
3. 初始化对象

如果线程 A 执行到步骤 2 时被挂起，线程 B 进入 `getInstance()`，第一次检查 `instance != null`（引用已指向内存），直接返回一个**未初始化的对象**，使用时会出错。

**解决方案**：加 `volatile` 禁止重排序

```java
private static volatile Singleton instance; // 加 volatile
```

**JMM 如何保证有序性？**
- `volatile`：通过**内存屏障**（Memory Barrier）禁止指令重排序
- `synchronized`：同一时刻只有一个线程执行，相当于单线程，天然有序
- **happens-before 规则**：JMM 定义的有序性保证规则

**happens-before 常见规则**：
1. **程序顺序规则**：同一线程中，前面的操作 happens-before 后面的操作
2. **volatile 规则**：volatile 写 happens-before 后续的 volatile 读
3. **锁规则**：unlock happens-before 后续的 lock
4. **线程启动规则**：`thread.start()` happens-before 子线程中的每个操作
5. **线程终止规则**：线程中的所有操作 happens-before `thread.join()` 返回
6. **传递性规则**：A happens-before B，B happens-before C，则 A happens-before C

### 解决方案总结

| 方案 | 原子性 | 可见性 | 有序性 | 适用场景 |
|------|:---:|:---:|:---:|------|
| `synchronized` | 保证 | 保证 | 保证 | 写多读少，一致性要求高 |
| `volatile` | 不保证 | 保证 | 保证 | 标志位、状态值 |
| `Lock`（ReentrantLock） | 保证 | 保证 | 保证 | 复杂并发场景 |
| 原子类（AtomicInteger） | 保证 | 保证 | 保证 | 简单计数/统计场景 |

**原子类**是最轻量的方案，底层依赖 CPU 的原子指令 `cmpxchg`，通过 Unsafe 类直接操作内存实现。适合简单的递增递减等场景，代码简洁且性能高。

```java
AtomicInteger requestCount = new AtomicInteger(0);
requestCount.incrementAndGet(); // 线程安全的原子递增
```

### 常考算法：多线程交替打印 0-100

**题目**：两个线程交替打印 0-100 的数字，线程 A 打印偶数，线程 B 打印奇数。

**方案一：synchronized + wait/notify**

```java
public class PrintNumber {
    private static int number = 0;
    private static final Object lock = new Object();

    public static void main(String[] args) {
        Thread evenThread = new Thread(() -> {
            while (number <= 100) {
                synchronized (lock) {
                    if (number % 2 == 0 && number <= 100) {
                        System.out.println(Thread.currentThread().getName() + ": " + number);
                        number++;
                        lock.notify();
                    }
                    if (number <= 100) {
                        try { lock.wait(); } catch (InterruptedException e) { e.printStackTrace(); }
                    }
                }
            }
        }, "偶数线程");

        Thread oddThread = new Thread(() -> {
            while (number <= 100) {
                synchronized (lock) {
                    if (number % 2 != 0 && number <= 100) {
                        System.out.println(Thread.currentThread().getName() + ": " + number);
                        number++;
                        lock.notify();
                    }
                    if (number <= 100) {
                        try { lock.wait(); } catch (InterruptedException e) { e.printStackTrace(); }
                    }
                }
            }
        }, "奇数线程");

        evenThread.start();
        oddThread.start();
    }
}
```

**方案二：ReentrantLock + Condition**

```java
public class PrintNumber {
    private static int number = 0;
    private static final ReentrantLock lock = new ReentrantLock();
    private static final Condition condition = lock.newCondition();

    public static void main(String[] args) {
        Thread evenThread = new Thread(() -> {
            while (number <= 100) {
                lock.lock();
                try {
                    while (number % 2 != 0 && number <= 100) {
                        condition.await();
                    }
                    if (number <= 100) {
                        System.out.println(Thread.currentThread().getName() + ": " + number);
                        number++;
                        condition.signal();
                    }
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }, "偶数线程");

        Thread oddThread = new Thread(() -> {
            while (number <= 100) {
                lock.lock();
                try {
                    while (number % 2 == 0 && number <= 100) {
                        condition.await();
                    }
                    if (number <= 100) {
                        System.out.println(Thread.currentThread().getName() + ": " + number);
                        number++;
                        condition.signal();
                    }
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }, "奇数线程");

        evenThread.start();
        oddThread.start();
    }
}
```

### 常考算法：多线程交替打印 ABC

**题目**：三个线程交替打印 A、B、C，输出 "ABCABCABC..."

**方案一：synchronized + wait/notify**

```java
public class PrintABC {
    private static int state = 0; // 0=打印A, 1=打印B, 2=打印C
    private static final Object lock = new Object();

    public static void main(String[] args) {
        new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                synchronized (lock) {
                    while (state % 3 != 0) {
                        try { lock.wait(); } catch (InterruptedException e) { e.printStackTrace(); }
                    }
                    System.out.print("A");
                    state++;
                    lock.notifyAll();
                }
            }
        }, "线程A").start();

        new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                synchronized (lock) {
                    while (state % 3 != 1) {
                        try { lock.wait(); } catch (InterruptedException e) { e.printStackTrace(); }
                    }
                    System.out.print("B");
                    state++;
                    lock.notifyAll();
                }
            }
        }, "线程B").start();

        new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                synchronized (lock) {
                    while (state % 3 != 2) {
                        try { lock.wait(); } catch (InterruptedException e) { e.printStackTrace(); }
                    }
                    System.out.print("C");
                    state++;
                    lock.notifyAll();
                }
            }
        }, "线程C").start();
    }
}
```

**方案二：ReentrantLock + 三个 Condition（精确唤醒）**

```java
public class PrintABC {
    private static int state = 0;
    private static final ReentrantLock lock = new ReentrantLock();
    private static final Condition conditionA = lock.newCondition();
    private static final Condition conditionB = lock.newCondition();
    private static final Condition conditionC = lock.newCondition();

    public static void main(String[] args) {
        new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                lock.lock();
                try {
                    while (state % 3 != 0) conditionA.await();
                    System.out.print("A");
                    state++;
                    conditionB.signal(); // 精确唤醒 B
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }, "线程A").start();

        new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                lock.lock();
                try {
                    while (state % 3 != 1) conditionB.await();
                    System.out.print("B");
                    state++;
                    conditionC.signal(); // 精确唤醒 C
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }, "线程B").start();

        new Thread(() -> {
            for (int i = 0; i < 10; i++) {
                lock.lock();
                try {
                    while (state % 3 != 2) conditionC.await();
                    System.out.print("C");
                    state++;
                    conditionA.signal(); // 精确唤醒 A
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }, "线程C").start();
    }
}
```

**方案三：AtomicInteger + 自旋**

```java
public class PrintABC {
    private static AtomicInteger state = new AtomicInteger(0);

    public static void main(String[] args) {
        new Thread(() -> {
            for (int i = 0; i < 10;) {
                if (state.get() % 3 == 0) {
                    System.out.print("A");
                    state.incrementAndGet();
                    i++;
                }
            }
        }, "线程A").start();

        new Thread(() -> {
            for (int i = 0; i < 10;) {
                if (state.get() % 3 == 1) {
                    System.out.print("B");
                    state.incrementAndGet();
                    i++;
                }
            }
        }, "线程B").start();

        new Thread(() -> {
            for (int i = 0; i < 10;) {
                if (state.get() % 3 == 2) {
                    System.out.print("C");
                    state.incrementAndGet();
                    i++;
                }
            }
        }, "线程C").start();
    }
}
```

**方案对比**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| synchronized + wait/notify | 简单直观 | notifyAll 唤醒所有线程，不够精确 |
| ReentrantLock + Condition | 精确唤醒，效率高 | 代码稍复杂 |
| AtomicInteger + 自旋 | 无锁，性能好 | 忙等消耗 CPU |
