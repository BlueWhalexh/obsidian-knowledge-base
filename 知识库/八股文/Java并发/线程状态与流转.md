# 线程状态与流转

## 问题
Java 线程有哪些状态？线程状态之间如何流转？创建线程有哪几种方式？

## 答案

### 6 种线程状态

Java 线程在 `java.lang.Thread.State` 枚举中定义了 6 种状态：

| 状态 | 说明 |
|------|------|
| **NEW** | 初始状态，线程被创建但尚未调用 `start()` 方法 |
| **RUNNABLE** | 可运行状态，包含就绪（Ready）和运行中（Running）两种子状态。调用 `start()` 后进入就绪态，获得 CPU 时间片后进入运行态 |
| **BLOCKED** | 阻塞状态，线程等待获取 synchronized 锁（监视器锁） |
| **WAITING** | 等待状态，线程进入无限期等待，需要被其他线程显式唤醒。进入方式：`Object.wait()`、`Thread.join()`、`LockSupport.park()` |
| **TIMED_WAITING** | 超时等待状态，线程在指定时间内等待，超时后自动唤醒。进入方式：`Thread.sleep(n)`、`Object.wait(n)`、`Thread.join(n)`、`LockSupport.parkNanos/parkUntil` |
| **TERMINATED** | 终止状态，线程执行完毕或因异常退出 |

### 状态流转图

```
                    start()
  NEW ──────────────────────→ RUNNABLE (Ready)
                                 │
                    获得CPU时间片    │ 时间片耗尽/yield()
                    ┌────────────┘
                    ↓               ↑
              RUNNABLE (Running) ──┘
                    │
        ┌───────────┼───────────────┐
        ↓           ↓               ↓
   synchronized   wait()/join()   sleep(n)/wait(n)
   锁竞争失败     park()          join(n)/parkNanos
        │           │               │
        ↓           ↓               ↓
    BLOCKED      WAITING        TIMED_WAITING
        │           │               │
   获得锁/唤醒    notify()/        超时/获得锁
        │       notifyAll()         │
        └─────→ RUNNABLE ←─────────┘
                    │
            run() 方法执行完毕/异常
                    ↓
               TERMINATED
```

**关键流转路径**：
- `NEW → RUNNABLE`：调用 `start()` 方法
- `RUNNABLE → BLOCKED`：尝试获取 synchronized 锁失败
- `BLOCKED → RUNNABLE`：获得锁
- `RUNNABLE → WAITING`：调用 `wait()`、`join()`、`park()`
- `WAITING → RUNNABLE`：被 `notify()`/`notifyAll()` 唤醒，或 `join()` 的目标线程结束
- `RUNNABLE → TIMED_WAITING`：调用 `sleep(n)`、`wait(n)` 等带超时的方法
- `TIMED_WAITING → RUNNABLE`：超时时间到，或被唤醒
- `RUNNABLE → TERMINATED`：`run()` 方法执行完毕或抛出未捕获异常

### 创建线程的 4 种方式

| 方式 | 说明 | 特点 |
|------|------|------|
| **继承 Thread 类** | 重写 `run()` 方法，调用 `start()` | Java 单继承限制，不推荐 |
| **实现 Runnable 接口** | 实现 `run()` 方法，传给 Thread 构造器 | 推荐，解耦任务与线程 |
| **实现 Callable 接口** | 实现 `call()` 方法，可返回结果和抛异常 | 支持返回值，配合 FutureTask 使用 |
| **线程池创建** | 通过 `ThreadPoolExecutor` 提交任务 | 推荐方式，复用线程，资源可控 |

### 上下文切换

上下文切换是指 CPU 从一个线程转到另一个线程时，需要保存当前线程的上下文状态，恢复另一个线程的上下文状态。

**触发条件**：
1. **时间片耗尽**：操作系统为每个线程分配固定的 CPU 时间片，用完后强制切换
2. **线程主动阻塞**：等待 I/O、获取锁失败、调用 `sleep()` 或 `wait()`
3. **高优先级线程抢占**：更高优先级线程进入就绪状态
4. **线程主动让出**：调用 `Thread.yield()`
5. **中断处理**：硬件事件（磁盘 I/O 完成、网络数据到达）引发中断

**减少上下文切换的方法**：
- 合理设置线程数，避免线程过多
- 使用无锁并发编程（CAS）
- 使用虚拟线程（协程），避免操作系统级别上下文切换
- 缩小同步块范围，减少线程等待时间
