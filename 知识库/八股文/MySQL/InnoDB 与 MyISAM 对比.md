# InnoDB 与 MyISAM 对比

## 问题
InnoDB 和 MyISAM 有什么区别？各自适用于什么场景？

## 答案

### 核心区别对比

| 特性 | InnoDB | MyISAM |
|------|--------|--------|
| **事务支持** | 支持（ACID） | 不支持 |
| **外键** | 支持 | 不支持 |
| **行级锁** | 支持 | 不支持（只有表级锁） |
| **聚簇索引** | 支持 | 不支持 |
| **全文索引** | MySQL 5.6+ 支持 | 支持 |
| **崩溃恢复** | 支持（redo log） | 不支持 |
| **默认引擎** | MySQL 5.5+ 默认 | MySQL 5.5 之前默认 |
| **存储文件** | .frm + .ibd | .frm + .MYD + .MYI |
| **MVCC** | 支持 | 不支持 |

### InnoDB 特点
- **事务安全**：支持完整的 ACID 事务，通过 undo log 实现回滚，redo log 实现崩溃恢复
- **行级锁**：并发性能好，适合高并发读写场景
- **聚簇索引**：数据按主键顺序存储，主键查询极快
- **外键约束**：保证数据的引用完整性
- **MVCC**：通过多版本并发控制实现非锁定读，提升并发性能

### MyISAM 特点
- **不支持事务**：操作不可回滚
- **表级锁**：写操作会锁整张表，并发写入性能差
- **全文索引**：较早版本就支持全文搜索
- **非聚簇索引**：索引和数据分开存储（.MYI 存索引，.MYD 存数据）
- **存储空间小**：支持压缩表

### 适用场景

**选 InnoDB**：
- 需要事务支持（如订单、支付等）
- 高并发读写场景
- 需要外键约束
- 数据安全性要求高

**选 MyISAM**：
- 以读操作为主的应用（如日志分析、数据仓库）
- 不需要事务
- 需要全文索引且 MySQL 版本较低
- 数据量较小、对写入并发要求不高

> MySQL 5.5 之后默认引擎为 InnoDB，现代业务开发基本都使用 InnoDB。
