# 查询关键字执行顺序

## 问题
SQL 查询语句中各关键字的执行顺序是什么？为什么 WHERE 在 GROUP BY 之前执行？

## 答案

### 执行顺序
SQL 查询关键字的**逻辑执行顺序**如下（与书写顺序不同）：

```
FROM → ON → JOIN → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT
```

### 各步骤详解

| 步骤 | 关键字 | 说明 |
|------|--------|------|
| 1 | **FROM** | 确定数据来源的表 |
| 2 | **ON** | 处理 JOIN 的关联条件 |
| 3 | **JOIN** | 合并关联的表数据 |
| 4 | **WHERE** | 对 JOIN 结果应用过滤条件，过滤行数据 |
| 5 | **GROUP BY** | 按指定字段分组 |
| 6 | **HAVING** | 筛选过滤分组后的数据 |
| 7 | **SELECT** | 选择需要返回的字段（计算列、别名等） |
| 8 | **DISTINCT** | 去除重复行 |
| 9 | **ORDER BY** | 对结果排序 |
| 10 | **LIMIT** | 限制返回行数 |

### 示例
```sql
SELECT name, COUNT(*)        -- 7
FROM employees               -- 1
JOIN departments              -- 3
  ON employees.dept_id = departments.id  -- 2
WHERE employees.salary > 50000  -- 4
GROUP BY departments.name    -- 5
HAVING COUNT(*) > 10         -- 6
ORDER BY name                -- 8
LIMIT 5;                     -- 9
```

### 关键理解点

**为什么 WHERE 在 GROUP BY 之前？**
- WHERE 是在分组前对**原始行数据**进行过滤
- GROUP BY 是对过滤后的数据进行分组
- 如果先分组再过滤，需要对更多数据执行分组操作，效率更低
- WHERE 不能使用聚合函数（因为此时还未分组），聚合函数过滤需要用 HAVING

**为什么 SELECT 在 GROUP BY 之后？**
- SELECT 中可以使用别名，而别名的定义依赖于之前的步骤
- GROUP BY 和 HAVING 中不能直接使用 SELECT 中定义的别名（MySQL 部分版本除外）
- SELECT 是在数据筛选、分组、过滤全部完成后，才选择最终返回的字段

**注意事项**：
- 以上是逻辑执行顺序，实际物理执行可能不同，优化器会根据统计信息、索引等因素调整执行计划
- 了解执行顺序有助于理解为什么某些写法会报错（如在 WHERE 中使用聚合函数）
