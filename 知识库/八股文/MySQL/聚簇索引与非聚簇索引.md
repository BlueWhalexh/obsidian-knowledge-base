# 聚簇索引与非聚簇索引

## 问题
什么是聚簇索引和非聚簇索引？它们有什么区别？什么是回表？什么是覆盖索引？

## 答案

### 聚簇索引（Clustered Index）
- **数据和索引存储在一起**。非叶子节点存储索引字段的值，叶子节点存储该条记录的**整行数据**
- 每个表**只能有一个**聚簇索引
- 在 InnoDB 中，**主键索引就是聚簇索引**，表数据按照主键顺序存储在磁盘上
- 物理存储顺序与主键逻辑顺序一致，因此按主键查找速度非常快

**默认行为**：
1. 如果显式定义了主键，则主键为聚簇索引
2. 如果未定义主键，优先使用第一个非空唯一索引作为聚簇索引
3. 如果都没有，InnoDB 自动生成隐藏的 `DB_ROW_ID` 作为聚簇索引

### 非聚簇索引（Secondary Index）
- **数据和索引分开存储**。非叶子节点存储索引值，叶子节点存储的是**主键值 + 索引值**
- 一个表可以有**多个**非聚簇索引（即二级索引）
- 不影响表中数据的物理存储顺序
- 根据非主键字段创建的索引都是非聚簇索引

### 回表查询
通过非聚簇索引查询时，如果 SELECT 的字段不在索引中，需要经历两步：
1. 先在非聚簇索引树上查到主键 ID
2. 再通过主键 ID 到聚簇索引树上查找完整的行数据

这个过程称为**回表**，会产生额外的磁盘 IO。

### 覆盖索引（Covering Index）
如果查询所需的字段**全部包含在索引中**，则无需回表，直接从索引中获取数据，称为覆盖索引。

```sql
-- 联合索引 (name, age)
-- 覆盖索引，无需回表
SELECT name, age FROM user WHERE name = 'Tom';

-- 需要回表，因为 email 不在索引中
SELECT name, age, email FROM user WHERE name = 'Tom';
```

使用 `EXPLAIN` 时，`Extra` 列显示 `Using index` 表示使用了覆盖索引。
