# 单线程 Redis 为什么那么快？

## 问题
Redis 是单线程的，为什么性能还那么高？单线程模型有什么优势？

## 答案

Redis 的瓶颈在 **网络 IO** 而不是 CPU 计算，单线程反而避免了上下文切换和锁竞争的开销。

### 核心原因

1. **基于内存操作**
   - 比磁盘快 10 万倍
   - 避免了磁盘 IO 瓶颈

2. **单线程模型**
   - 避免了多线程的上下文切换开销
   - 避免了锁竞争
   - 代码实现更简单，维护更容易

3. **IO 多路复用（epoll）**
   - 单线程能处理大量并发连接
   - 非阻塞 IO，事件驱动

4. **优化的数据结构**
   - **SDS（简单动态字符串）**：O(1) 获取长度
   - **跳跃表（ZSet）**：O(logN) 查询
   - **渐进式 rehash（Hash）**：不阻塞主线程

### 项目中使用场景

- 用户信息缓存（减少数据库查询）
- 组织权限缓存（权限校验加速）
- 会话存储（用户对话历史）
- Bitmap（文件分片上传状态）

## 补充：Redis 6.0 多线程

Redis 6.0 引入了 **IO 多线程**，但命令执行仍是单线程：
- 多线程只用于网络数据的读写和协议解析
- 命令执行还是在主线程串行进行
- 这样既利用了多核处理网络 IO，又保持了单线程执行的原子性
