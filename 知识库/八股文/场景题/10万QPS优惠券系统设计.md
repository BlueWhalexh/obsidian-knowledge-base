# 10万QPS优惠券系统设计

## 问题

如何设计支撑10万QPS的优惠券领取系统？如何解决热点库存和Redis超时问题？

## 面试考察点

- 热点库存扣减方案（子库存拆分）
- 二级缓存架构（Redis + 本地缓存）
- 高并发写入性能优化
- 消息队列异步化

## 解题思路

优惠券系统核心难点是**热点库存并发扣减**和**Redis超时降级**。采用子库存拆分 + 二级缓存方案。

## 具体方案

### 技术栈
Kitex(RPC框架) + MySQL + Redis + RocketMQ

### 1. 子库存解决热点问题
- **问题**：单个优惠券库存key被10万QPS集中访问，形成热点
- **方案**：将库存拆分为N个**子库存**（如100个子key）
- 领取时随机选择一个子库存扣减
- **随机不重复轮询**：遍历子库存列表，找到有库存的扣减

### 2. 二级缓存解决Redis超时
- **L1 本地缓存**：Caffeine/Guava，TTL极短（秒级）
- **L2 Redis缓存**：分布式缓存
- Redis超时时降级读本地缓存，保障可用性
- 写入时先更新Redis，异步刷新本地缓存

### 3. 库存扣减流程
```
1. 请求到达，先查本地缓存判断是否还有库存
2. 有库存则随机选择子库存，Lua脚本原子扣减
3. 扣减成功，发送MQ消息异步记录领取记录
4. 扣减失败（子库存为空），尝试下一个子库存
```

### 4. MQ异步化
- 扣减成功后不直接写DB，通过RocketMQ异步落库
- 削峰填谷，保护数据库

## 要点总结

1. 子库存拆分是解决热点库存的核心手段，分散单key压力
2. 随机不重复轮询保证子库存均匀消耗
3. 二级缓存（本地+Redis）解决Redis单点超时问题
4. MQ异步落库保护数据库，削峰填谷
