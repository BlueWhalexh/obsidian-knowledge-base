# CPU使用率高排查思路

## 问题
线上服务器CPU使用率飙升，如何定位是哪个进程、哪个线程、哪行代码导致的？

## 面试考察点
- Linux性能分析命令的熟练使用（top、printf、jstack）
- 从进程到线程到代码的逐层定位能力
- 对常见CPU高占用原因的判断（死循环、频繁GC、锁竞争、重复初始化）

## 排查思路
> top定位进程 -> top -Hp定位线程 -> printf转16进制 -> jstack查线程栈 -> 定位问题代码

## 具体排查步骤
1. **top命令定位进程**：`top`查看CPU占用最高的进程PID
2. **top -Hp定位线程**：`top -Hp <pid>`查看该进程中CPU最高的线程TID
3. **线程ID转16进制**：`printf %x <tid>`得到16进制nid
4. **jstack查线程栈**：`jstack <pid> | grep -A 200 <nid>`查看线程堆栈
5. **分析堆栈定位代码**：找到RUNNABLE状态的线程，定位到具体代码行

## 常见原因与解决方案
| 原因 | 解决方案 |
|------|---------|
| 对象初始化放在方法内（如Validator每次new） | 将耗时对象初始化提到类级别，只创建一次 |
| 死循环/无限递归 | 检查循环退出条件 |
| 频繁FullGC | 参见FullGC排查 |
| 正则表达式回溯 | 优化正则表达式 |
| 序列化/反序列化大量数据 | 分批处理或优化序列化方式 |

## 要点总结
- 也可用Arthas工具：`thread -n 3`直接查看CPU占比前三的线程
- 关键命令组合：top -> top -Hp -> printf %x -> jstack | grep
- 线程状态为RUNNABLE且堆栈在业务代码中，基本就是问题所在
- 压测时关注CPU指标，QPS上升后CPU急剧升高通常说明有性能瓶颈
