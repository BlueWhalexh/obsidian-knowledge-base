# 千万级流量架构设计

## 问题

如何设计支撑千万级流量的系统架构？前端和后端分别需要做哪些优化？

## 面试考察点

- 前端优化（减少请求/页面静态化/边缘计算）
- 后端分层架构（基础设施/网络/架构层）
- 高性能设计四步骤
- 横向扩展能力

## 解题思路

千万级流量架构需要**前端减压 + 后端分层 + 高可用设计**。

## 具体方案

### 1. 前端优化（减少到达后端的请求量）
- **减少请求**：合并CSS/JS、雪碧图、懒加载
- **页面静态化**：静态页面部署到CDN
- **边缘计算**：在CDN节点执行部分逻辑（如鉴权、AB测试）

### 2. 后端分层

**基础设施层**
- 负载均衡（Nginx/LVS/F5）
- DNS轮询
- 多机房部署

**网络层**
- CDN加速静态资源
- TCP优化（连接池、Keep-Alive）
- HTTP/2多路复用

**架构层**
- 微服务拆分，独立扩缩容
- 多级缓存（本地+分布式+DB）
- 消息队列异步化
- 数据库分库分表

### 3. 高性能设计四步骤
1. **明确指标**：QPS、RT、可用性目标
2. **保护系统**：限流、熔断、降级
3. **用户体验**：核心链路优先保障
4. **快速扩容**：容器化、自动扩缩容

## 要点总结

1. 前端优化能过滤50%以上的请求，是最高效的优化手段
2. 页面静态化+CDN是读多写少场景的必备方案
3. 后端分层架构，每层独立扩展
4. 四步骤方法论：明确指标 -> 保护系统 -> 体验优先 -> 快速扩容
