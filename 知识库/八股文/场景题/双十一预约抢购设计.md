# 双十一预约抢购设计

## 问题

如何设计双十一预约抢购系统？如何应对瞬时超高并发？

## 面试考察点

- 四阶段流程设计（预约/等待/抢购/支付）
- Redis分布式锁 + Lua脚本原子操作
- 页面静态化 + CDN
- MQ削峰
- Redis缓存库存

## 解题思路

抢购系统的核心挑战是**瞬时高并发写入**和**库存超卖防控**。采用四阶段流程分层削峰。

## 具体方案

### 四阶段设计

**阶段一：预约**
- 用户提前预约，记录预约信息
- 预约数据写入数据库，量可控

**阶段二：等待**
- 页面静态化 + CDN分发
- 减少服务器压力，用户看到的是静态页面

**阶段三：抢购**
- 核心高并发阶段
- **Redis缓存库存**：库存预加载到Redis
- **Lua脚本原子扣减**：`if redis.call('get',KEY) > 0 then redis.call('decr',KEY) return 1 else return 0 end`
- **分布式锁**：防止同一用户重复下单
- **MQ削峰**：扣减成功后发MQ异步创建订单

**阶段四：支付**
- MQ消费订单消息，创建订单
- 用户支付，支付成功扣减数据库库存
- 超时未支付，回滚Redis库存

### 关键设计点
- 请求层层过滤：CDN -> Nginx限流 -> Redis判断 -> MQ -> DB
- 最终到达数据库的请求量远小于原始流量

## 要点总结

1. 四阶段流程将流量分层削峰，每层过滤大量请求
2. Redis Lua脚本保证库存扣减的原子性
3. 页面静态化+CDN承担绝大部分读流量
4. MQ异步创建订单，保护数据库
5. 层层过滤后到达DB的请求量级大幅降低
