# 大文件排序

## 问题

有一个 10GB 的文件，里面每行是一个整数，内存只有 1GB，如何对文件进行排序？

## 方案一：外部排序（External Sort）

### 核心思想

分治：将大文件拆分成多个小文件，每个小文件可以放入内存排序，然后将排序好的小文件进行多路归并。

### 详细步骤

#### 第一步：文件拆分 + 内存排序（生成初始归并段）

```
1. 将 10GB 文件分成 10 个 1GB 的小文件（chunk_0 ~ chunk_9）
2. 逐个读入内存，使用快速排序/堆排序
3. 将排序结果写回磁盘，生成 10 个有序文件
```

#### 第二步：多路归并（K-Way Merge）

```
1. 为每个有序文件打开一个文件指针
2. 读取每个文件的第一条数据放入小顶堆（堆大小 = 文件数 = 10）
3. 取出堆顶元素写入结果文件
4. 从堆顶元素所属的文件中读取下一个数据放入堆
5. 重复步骤 3-4 直到所有文件读取完毕
```

### 代码实现

```java
public class ExternalSort {
    
    private static final int MEMORY_SIZE = 1024 * 1024; // 1GB模拟
    
    /**
     * 第一步：生成初始归并段
     */
    public List<File> createInitialRuns(File inputFile) throws IOException {
        List<File> runs = new ArrayList<>();
        
        try (BufferedReader reader = new BufferedReader(
                new FileReader(inputFile))) {
            List<Long> buffer = new ArrayList<>(MEMORY_SIZE);
            String line;
            
            while ((line = reader.readLine()) != null) {
                buffer.add(Long.parseLong(line));
                
                if (buffer.size() >= MEMORY_SIZE) {
                    // 内存排序
                    Collections.sort(buffer);
                    // 写入临时文件
                    File runFile = writeToFile(buffer);
                    runs.add(runFile);
                    buffer.clear();
                }
            }
            
            // 处理剩余数据
            if (!buffer.isEmpty()) {
                Collections.sort(buffer);
                runs.add(writeToFile(buffer));
            }
        }
        
        return runs;
    }
    
    /**
     * 第二步：多路归并
     */
    public void mergeRuns(List<File> runs, File outputFile) 
            throws IOException {
        // 小顶堆：元素为 (值, 文件索引)
        PriorityQueue<long[]> minHeap = new PriorityQueue<>(
            runs.size(), Comparator.comparingLong(a -> a[0]));
        
        // 打开所有文件的 BufferedReader
        BufferedReader[] readers = new BufferedReader[runs.size()];
        for (int i = 0; i < runs.size(); i++) {
            readers[i] = new BufferedReader(new FileReader(runs.get(i)));
            String line = readers[i].readLine();
            if (line != null) {
                minHeap.offer(new long[]{Long.parseLong(line), i});
            }
        }
        
        // 归并输出
        try (BufferedWriter writer = new BufferedWriter(
                new FileWriter(outputFile))) {
            while (!minHeap.isEmpty()) {
                long[] min = minHeap.poll();
                writer.write(min[0] + "\n");
                
                int fileIdx = (int) min[1];
                String nextLine = readers[fileIdx].readLine();
                if (nextLine != null) {
                    minHeap.offer(new long[]{
                        Long.parseLong(nextLine), fileIdx});
                }
            }
        }
        
        // 关闭所有 reader
        for (BufferedReader reader : readers) {
            reader.close();
        }
    }
    
    private File writeToFile(List<Long> data) throws IOException {
        File tempFile = File.createTempFile("run_", ".tmp");
        try (BufferedWriter writer = new BufferedWriter(
                new FileWriter(tempFile))) {
            for (Long value : data) {
                writer.write(value + "\n");
            }
        }
        return tempFile;
    }
}
```

### 时间复杂度分析

假设文件大小 N，内存大小 M，分成 K = N/M 个归并段：

- 第一步（拆分排序）：每段排序 O(M logM)，共 K 段 → O(N logM)
- 第二步（多路归并）：每条数据入堆出堆一次 O(logK)，共 N 条 → O(N logK)
- 总计：O(N logM + N logK) = O(N logN)
- 磁盘 IO：每条数据读写各 2 次 → 4N 次 IO

### 优化：使用置换选择排序（Replacement Selection）

生成长度大于内存的初始归并段，减少归并路数。

```
1. 用 M 大小的最小堆构建初始堆
2. 不断输出堆顶到当前归并段
3. 新元素如果 >= 上一个输出值，插入堆
4. 新元素如果 < 上一个输出值，标记为"下一个归并段"
5. 当前堆清空后，开始下一个归并段
```

最优情况下（数据基本有序），可以只生成 1 个归并段，无需归并。

## 方案二：位图法（Bitmap Sort）

### 适用条件

- 数据范围已知且不太大（如 1 到 10 亿）
- 数据不重复（或允许去重）
- 需要排序（而非仅仅查找 Top K）

### 核心思想

用一个 bit 数组来表示某个数是否存在。第 i 位为 1 表示数字 i 存在。

### 空间计算

```
10 亿个整数，范围 0 ~ 10 亿
位图大小 = 10 亿 bit = 125MB（远小于 1GB 内存限制）
```

### 代码实现

```java
public class BitmapSort {
    
    /**
     * 方式1：直接使用 BitSet
     */
    public void sortWithBitSet(File inputFile, File outputFile) 
            throws IOException {
        BitSet bitSet = new BitSet(1_000_000_000);
        
        // 读取数据并设置位
        try (BufferedReader reader = new BufferedReader(
                new FileReader(inputFile))) {
            String line;
            while ((line = reader.readLine()) != null) {
                int num = Integer.parseInt(line.trim());
                bitSet.set(num);
            }
        }
        
        // 按顺序输出
        try (BufferedWriter writer = new BufferedWriter(
                new FileWriter(outputFile))) {
            int idx = -1;
            while ((idx = bitSet.nextSetBit(idx + 1)) >= 0) {
                writer.write(idx + "\n");
            }
        }
    }
    
    /**
     * 方式2：手动使用 byte 数组（更底层）
     */
    public void sortWithByteArray(int[] data) {
        // 假设数据范围 0 ~ 10 亿
        int bitmapSize = 1_000_000_000 / 8 + 1; // 约 125MB
        byte[] bitmap = new byte[bitmapSize];
        
        // 设置位
        for (int num : data) {
            bitmap[num / 8] |= (1 << (num % 8));
        }
        
        // 输出排序结果
        List<Integer> sorted = new ArrayList<>();
        for (int i = 0; i < bitmapSize; i++) {
            for (int j = 0; j < 8; j++) {
                if ((bitmap[i] & (1 << j)) != 0) {
                    sorted.add(i * 8 + j);
                }
            }
        }
    }
}
```

### 时间复杂度

- 时间：O(N)，只需遍历数据两次（一次读入，一次输出）
- 空间：O(数据范围 / 8) 字节
- 是最快的排序方案，但限制条件多

## 方案对比

| 方案 | 时间复杂度 | 空间复杂度 | 适用条件 |
|------|-----------|-----------|---------|
| 外部排序 | O(N logN) | O(M) M=内存大小 | 通用，无特殊限制 |
| 位图法 | O(N) | O(数据范围/8) | 数据范围已知、不重复 |
| 直接内存排序 | O(N logN) | O(N) | 数据能放入内存 |

## 扩展：大规模数据处理常见问题

### 1. 两个大文件求交集

**问题**：文件 A 有 10 亿个 URL，文件 B 有 10 亿个 URL，求交集。

**方案**：
- **位图法**：如果 URL 可以哈希到固定范围，用两个位图
- **分桶法**：对 URL 取哈希值 mod N，分到 N 个桶中，再对每个桶求交集
- **布隆过滤器**：允许一定误判率时使用

### 2. 大文件中找重复行

```
1. 计算每行的哈希值（如 MD5）
2. 将哈希值相同的行分到同一个桶
3. 在每个桶内精确比较
```

### 3. 海量数据 Top K

参考 [[Top K 问题]]，使用分治 + 小顶堆方案。

## 常见追问

### Q：外部排序如何优化磁盘 IO？

> 1. 使用大缓冲区（BufferedInputStream），减少系统调用次数
> 2. 使用 SSD 替代 HDD，随机读写性能提升百倍
> 3. 并行读写：一个线程读数据，一个线程排序，一个线程写数据（流水线）
> 4. 使用内存映射文件（MappedByteBuffer），减少用户态/内核态切换

### Q：如果数据有重复怎么办？

> 位图法只能标记存在与否，无法统计重复次数。此时可以使用：
> - **计数布隆过滤器**
> - **桶排序 + 哈希分桶**，每个桶内统计频次后再排序

### Q：分布式环境下如何排序？

> 1. 对数据按 key 范围进行分区（Range Partition），每个分区交给一台机器排序
> 2. 或者对数据取哈希值分区（Hash Partition）
> 3. 每台机器独立排序
> 4. 最后合并各分区结果（分区之间天然有序）
