# 布隆过滤器内存估算

## 问题
5亿条数据放到布隆过滤器中，大概需要多大内存？如何估算？

## 面试考察点
- 布隆过滤器原理的深入理解（哈希函数、误判率）
- 大数据量下的内存估算能力（工程数学应用）
- 空间-精度权衡意识（如何选择合理参数）
- 分布式系统设计思维（海量数据场景的优化）

## 解题思路
> 核心公式：位数组大小 m = -n*ln(p) / (ln2)^2，哈希函数数量 k = (m/n)*ln2，其中n为数据量，p为误判率。

5亿数据不同误判率下的内存需求：1%误判 -> 0.56GB（7个哈希函数），0.1% -> 0.85GB（10个），0.01% -> 1.12GB（14个），0.001% -> 1.4GB（17个）。

估算步骤（以1%误判为例）：n=5x10^8，p=0.01，计算m = -(5x10^8 * ln0.01) / (ln2)^2 = 4.8x10^9 bit，换算 = 4.8x10^9 / 8 / 1024^3 = 0.56GB，哈希函数k = 4.8x10^9 / 5x10^8 * ln2 = 7。

## 具体方案
### 方案一：标准布隆过滤器
- Guava实现：`BloomFilter.create(Funnels.stringFunnel(), 500_000_000, 0.01)`
- Redis实现：`BF.RESERVE my_filter 0.01 500000000`（RedisBloom模块）
- 优点：空间效率高，查询O(k)
- 缺点：不支持删除，位数组不可动态扩容

### 方案二：布谷鸟过滤器（Cuckoo Filter）
- 支持删除操作，内存约1.2-1.5x布隆过滤器
- 适合需要动态更新的场景（缓存失效、黑名单管理）

### 方案三：动态布隆过滤器（Scalable Bloom Filter）
- 支持动态扩容，避免预估内存不足
- 适合数据量不确定的场景

## 要点总结
- 核心公式：m = -n*ln(p)/(ln2)^2，k = -ln(p)/ln2
- 5亿数据1%误判率需约0.56GB内存，7个哈希函数
- 速记：每降低10倍误判率，内存增加约50%
- 位数组填充到约50%时误判率最低，接近100%时急剧上升
- 不支持删除操作，需删除场景使用布谷鸟过滤器；哈希函数推荐MurmurHash3/xxHash
