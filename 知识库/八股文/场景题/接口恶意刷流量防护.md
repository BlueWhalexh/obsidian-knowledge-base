# 接口恶意刷流量防护

## 问题
防止接口被恶意刷流量，除了限流还应在代码层面做哪些防护？

## 面试考察点
- 攻击类型识别：SQL注入、XSS、暴力破解、参数篡改
- 防御性编程：输入参数校验、安全编码规范
- 攻击拦截策略：黑名单机制、行为分析
- 交互式验证与签名验证的实现

## 解题思路
> 核心思路：限流只是防护手段之一，还需从交互式验证、安全参数校验、HTTPS、用户认证、资源授权、IP封禁、日志监控等多维度构建防护体系。

接口被恶意刷流量的严重后果：自动化工具针对高频接口（注册登录、秒杀抢购）发起攻击，瞬时吞吐量可达1W-10W QPS，导致系统雪崩甚至瘫痪。防护措施按优先级递进：验证码/人机验证（防自动化）-> 安全参数校验（防伪造请求）-> HTTPS（防窃听篡改）-> 用户认证（Token验证）-> 资源授权（RBAC权限控制）-> IP封禁 -> 日志监控分析。

安全参数校验的实现：请求中添加时间戳、随机字符串、签名，服务端按相同算法计算签名值比对。适用于防低强度攻击，高强度攻击需配合用户认证和IP白名单。

## 具体方案
### 方案一：交互式验证+参数签名
- 验证码/滑块验证防自动化攻击
- 参数签名：timestamp + nonce + 参数排序拼接 + 密钥 -> MD5签名
- 适用于防低强度垃圾请求和非法爬虫

### 方案二：用户认证+RBAC授权
- Token认证：登录后生成Token，请求时在Header中携带验证
- RBAC访问控制：基于角色的权限管理，精准控制资源访问
- Shiro框架实现身份验证和授权

### 方案三：WAF+行为分析
- Nginx WAF配置：拦截SQL注入（union|select|drop）、XSS（<script>|onerror）
- 行为分析：设备指纹+IP聚合识别脚本自动化攻击
- 黑名单机制：高频失败登录自动封禁IP

## 要点总结
- 除限流外还需：交互式验证、安全参数校验、HTTPS、用户认证、资源授权、IP封禁、日志监控
- 参数签名方案可防低强度攻击，高强度攻击需配合Token认证和IP白名单
- 对敏感接口（短信验证码、登录注册）必须使用HTTPS
- WAF可在Nginx层拦截SQL注入和XSS攻击
- 行为分析（设备指纹+IP聚类）可识别脚本自动化攻击
