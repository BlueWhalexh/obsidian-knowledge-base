# 搜索引擎设计避免大海捞针

## 问题

如何设计一个高效的搜索引擎？从索引构建到查询优化的完整链路是什么？

## 面试考察点

- 正排索引 vs 倒排索引
- 索引构建性能优化（64桶并行）
- 倒排链表求交集算法（拉链法、跳表优化）
- PageRank排序算法

## 解题思路

搜索引擎核心是**倒排索引** + **高效求交** + **相关性排序**。

## 具体方案

### 1. 正排索引 -> 倒排索引
- **正排索引**：文档ID -> 文档内容（适合按ID查内容）
- **倒排索引**：关键词 -> 文档ID列表（适合按关键词查文档）
- 倒排索引是搜索引擎的基石

### 2. 64桶并行构建索引
- 将文档按hash分到64个桶
- 每个桶独立构建倒排索引
- 64个线程并行处理，大幅提升索引构建速度

### 3. 拉链法求交集
- 多个关键词的倒排链表求交集
- 双指针遍历，时间复杂度 O(2n)
- 适用于两个链表长度接近的场景

### 4. 跳表优化
- 在倒排链表上建立多级跳表
- 求交集时跳过不匹配的区间
- 时间复杂度优化到 O(logn)
- 适用于链表长度差异大的场景

### 5. PageRank排序
- 基于网页链接关系计算重要性
- **阻尼因子 α = 0.85**：用户有85%概率点击链接，15%随机跳转
- 迭代计算直到收敛
- PR(A) = (1-α) + α * Σ(PR(Ti)/C(Ti))

## 要点总结

1. 倒排索引是搜索引擎的核心数据结构，关键词到文档ID的映射
2. 64桶并行构建索引，利用多线程加速
3. 拉链法O(2n)求交集，跳表优化到O(logn)
4. PageRank通过链接关系计算网页重要性，阻尼因子0.85是经典值
5. 整体链路：分词 -> 倒排索引 -> 求交集 -> PageRank排序 -> 返回结果
