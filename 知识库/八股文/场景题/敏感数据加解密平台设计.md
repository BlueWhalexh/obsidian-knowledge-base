# 敏感数据加解密平台设计（Venus）

## 问题

如何设计一个敏感数据加解密平台？如何安全存储和管理密钥？

## 面试考察点

- 密钥分片存储（异构物理隔离）
- SDK本地缓存密钥优化
- 领域模型设计（KeyBox/KeySuit/KeyChain/KeyChip）
- 密钥版本管理与轮转

## 解题思路

加解密平台核心是**密钥的安全管理**。密钥泄露等同于数据泄露，因此需要多层防护。

## 具体方案

### 1. 密钥分片存储
- 密钥不完整存储在任何单一位置
- **异构物理隔离**：不同分片存储在不同类型的系统中
  - 一部分在配置中心
  - 一部分在数据库
  - 一部分在KMS（密钥管理服务）
- 攻破任何一个系统都无法获取完整密钥

### 2. SDK本地缓存
- 每次加解密都请求远程密钥服务会很慢
- SDK本地缓存密钥，定期刷新
- 减少网络调用，提升加解密性能

### 3. 领域模型
- **KeyBox**：密钥容器，存储一个密钥的所有版本
- **KeySuit**：密钥套装，包含多个相关密钥
- **KeyChain**：密钥链，按业务场景串联密钥
- **KeyChip**：密钥芯片，最小粒度的密钥单元

### 4. 密钥版本管理
- 支持密钥轮转（定期更换密钥）
- 新数据用新密钥加密，旧数据仍可用旧密钥解密
- 版本号关联，解密时自动选择正确版本

## 要点总结

1. 密钥分片+异构物理隔离是核心安全策略，单点攻破不泄露
2. SDK本地缓存减少远程调用，提升性能
3. KeyBox/KeySuit/KeyChain/KeyChip四层领域模型清晰抽象
4. 密钥版本管理支持轮转，新旧密钥共存保障平滑过渡
