# 敏感词过滤

## 问题
如何设计一个高性能的敏感词过滤系统？如何应对谐音、拼音、组合词等变体绕过？

## 面试考察点
- 多模式匹配算法的选型（AC自动机 vs WM vs Trie树）
- 中文场景的特殊处理（拼音匹配、多音字、组合敏感词）
- 高性能场景下的吞吐量与内存权衡

## 答案

### 审核流程
完整的文本审核流程包含四个环节：名单匹配 -> 敏感词匹配 -> AI机器审核 -> 人工审核。敏感词匹配是核心环节，算法平均耗时50ms。

### 算法选型：AC自动机

| 特性 | AC自动机 | WM算法 |
|------|---------|--------|
| 优势 | 匹配效率高，模式串长度不影响效率 | 内存占用少，处理流程简单 |
| 劣势 | 内存占用大，加载耗时长 | 对模式串长度有要求（最小长度限制） |
| 适用 | 词库量大、长度不固定的场景 | 模式串长度较统一的场景 |

选AC自动机的原因：词库量大（百万级）、敏感词长度无法统一约定、加载不频繁且服务器内存充足。

### AC自动机原理
- **Trie字典树**：将所有敏感词构建成树结构，每个节点存储一个字符
- **Fail指针**：类似KMP的next数组，匹配失败时跳转到最长后缀节点继续匹配
- 匹配过程：按字符遍历目标串，在Trie树上转移，命中节点即为敏感词命中

### 组合敏感词匹配
某些词单独出现不违规，组合出现才违规（如"澳门"+"博彩"+"网站"）。

实现方案：
1. 将组合敏感词分割为单个词，维护词与组合的映射关系
2. 将单个词添加到AC自动机
3. 匹配后遍历结果，根据映射关系标记组合命中情况
4. 只有组合内所有词都命中，才判定该组合敏感词命中

### 拼音敏感词匹配
用户用谐音/同音字规避检测（如"啋票"代替"彩票"）。

实现方案：
- 将中文文本转换为拼音后再匹配，Trie树节点由char改为String（按音节匹配）
- 多音字处理：不依赖算法识别读音，而是将所有可能读音都列出，用DFS遍历拼音图的所有路径
- 拼音图是一维扩展到二维的"图"结构，配合DFS+回溯保证Trie树状态同步

### 性能优化要点
- 敏感词库支持动态热更新
- 内存敏感场景使用压缩Trie树
- 并发安全设计（线程安全的状态机构建）
- 分布式场景下敏感词库一致性保障

## 要点总结
- AC自动机是敏感词匹配的主流算法，O(n)匹配速度，适合百万级词库
- 组合敏感词通过映射关系+全部命中判定解决上下文语义问题
- 拼音敏感词通过拼音图+DFS遍历解决谐音/同音字绕过问题
- 实际系统通常组合使用：敏感词匹配 + AI审核 + 人工审核
