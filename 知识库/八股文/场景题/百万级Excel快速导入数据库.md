# 百万级Excel快速导入数据库

## 问题

如何将百万级数据的Excel快速导入数据库？有哪些优化手段？

## 面试考察点

- Excel读取框架选型（EasyExcel vs POI）
- 数据库查询缓存优化
- 批量插入优化（values批量 + 多线程）
- 日志对性能的影响

## 解题思路

Excel导入性能瓶颈在**数据读取**、**校验查询**和**数据库写入**三个环节。

## 具体方案

### 1. 第一版：POI + 逐行查询 + 逐行插入（最慢）
- 原生POI读取Excel，代码冗余
- 每行都查询数据库校验 -> N倍网络IO
- 逐行插入 -> N倍写入IO
- 适用于个位数/十位数数据量

### 2. 第二版：EasyExcel + 缓存查询 + 批量插入

**缓存校验数据**
- 将校验需要的数据一次性查出来存入HashMap
- 后续校验直接查内存，避免重复查询DB
- 空间换时间

**批量插入**
- MySQL支持 `INSERT INTO ... VALUES (),(),()` 一次插入多行
- Mybatis foreach拼接批量SQL
- **每次1000条**是最佳批量大小（过长SQL超过内存阈值会触发磁盘交换）

**EasyExcel替代POI**
- 阿里开源，基于SAX模式解析
- 内存占用极低（不会OOM）
- API简洁

### 3. 第三版：并行流批量插入
- 使用Java并行流 `parallelStream` 并发插入
- 利用CPU多核，榨干数据库CPU
- 16线程并行，10万行Excel导入总时间约95秒

### 4. 其他优化
- **避免循环中打日志**：info日志打印+磁盘持久化，耗时增加10倍
- 自定义 `InsertConsumer` 工具类封装并行插入逻辑
- 调整 `ForkJoinPool.common.parallelism` 控制并发数

## 要点总结

1. EasyExcel替代POI，内存占用低且API简洁
2. 校验数据缓存到HashMap，空间换时间避免N次DB查询
3. 批量插入每次1000条是最佳实践
4. 并行流多线程利用CPU多核，榨干数据库处理能力
5. 循环中避免打info日志，性能影响可达10倍
