# 蓝绿发布实现方案

## 问题

如何实现蓝绿发布？如何在不停机的情况下平滑切换新旧版本？

## 面试考察点

- Dubbo Tag路由机制
- 流量染色（ThreadLocal + Filter传递）
- 三套环境设计（无标签+蓝+绿）
- 兼容未接入蓝绿的应用

## 解题思路

蓝绿发布核心是**流量路由**：将不同流量导到不同版本的服务实例。

## 具体方案

### 1. 三套环境
- **无标签环境**：未接入蓝绿发布的普通服务
- **蓝环境（Blue）**：当前稳定版本
- **绿环境（Green）**：新版本，待验证

### 2. 流量染色
- 在请求入口（网关/Filter）设置**ThreadLocal标记**
- 标记当前请求属于蓝流量还是绿流量
- 通过**Filter链**在微服务间传递标记（HTTP Header / RPC Context）

### 3. Dubbo Tag路由
- Dubbo支持**tag路由**，根据tag将请求路由到对应分组
- Provider注册时携带tag标签
- Consumer请求时携带tag，Dubbo自动路由

### 4. 兼容策略
- 未接入蓝绿的服务标记为"无标签"
- 无标签流量可以路由到蓝或绿环境
- 逐步接入，不影响存量服务

### 5. 发布流程
1. 部署新版本到绿环境
2. 将测试流量导入绿环境验证
3. 验证通过后，逐步将生产流量从蓝切换到绿
4. 蓝环境保留，随时可回滚

## 要点总结

1. 流量染色是蓝绿发布的基础设施，ThreadLocal + Filter传递标记
2. Dubbo tag路由天然支持按标签分流
3. 三套环境（无标签/蓝/绿）兼容未接入的存量服务
4. 蓝环境保留可随时回滚，降低发布风险
