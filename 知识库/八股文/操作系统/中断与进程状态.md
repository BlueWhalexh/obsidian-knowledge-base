# 中断与进程状态

## 问题
什么是中断？什么是孤儿进程和僵尸进程？用户态和内核态有什么区别？进程调度算法有哪些？

## 答案

### 中断（Interrupt）

#### 什么是中断

中断是指计算机在执行程序过程中，出现了某些紧急事件（如硬件请求、程序错误），CPU 暂停当前正在执行的程序，转去处理该事件，处理完后再返回原来程序继续执行的过程。

#### 中断的分类

**1. 外部中断（硬件中断）**
- 由外部设备触发，与当前执行的指令无关
- **可屏蔽中断**：可以通过中断控制器屏蔽（如键盘、鼠标、网卡、磁盘）
- **不可屏蔽中断（NMI）**：无法屏蔽，必须立即处理（如电源故障、内存校验错误）

**触发方式**：
```
硬件设备 ──> 中断控制器（PIC/APIC） ──> CPU
                                          |
                                          v
                              CPU 暂停当前程序
                              保存现场（PC、寄存器）
                              跳转到中断处理程序
                              执行中断处理程序
                              恢复现场
                              继续执行原程序
```

**2. 内部中断（异常/软件中断）**
- 由 CPU 内部事件触发
- **陷阱（Trap）**：有意触发的中断（如系统调用 `int 0x80`）
- **故障（Fault）**：可恢复的错误（如缺页异常）
- **终止（Abort）**：不可恢复的错误（如除零错误、硬件故障）

**系统调用的过程**：
```
用户程序调用 read() 系统调用
       |
       v
触发 int 0x80（或 syscall 指令）
       |
       v
CPU 从用户态切换到内核态
       |
       v
保存用户态现场（寄存器、栈指针等）
       |
       v
查找系统调用表，跳转到对应的处理函数
       |
       v
执行内核代码（如从磁盘读取数据）
       |
       v
恢复用户态现场
       |
       v
返回用户程序，继续执行
```

#### 中断的作用

1. **提高 CPU 利用率**：CPU 不需要轮询设备状态，设备就绪时主动通知 CPU
2. **实现实时响应**：紧急事件可以打断当前程序立即处理
3. **实现系统调用**：用户程序通过中断请求内核服务
4. **实现进程调度**：时钟中断触发进程切换

### 孤儿进程（Orphan Process）

#### 定义

父进程退出后，子进程仍在运行，这些子进程就成为**孤儿进程**。

#### 产生过程

```
父进程 PID=100
  |
  ├── 子进程 PID=101
  └── 子进程 PID=102

父进程退出（PID=100 终止）
  |
  ├── 子进程 PID=101 （成为孤儿进程）
  └── 子进程 PID=102 （成为孤儿进程）
  |
  v
init 进程（PID=1）收养这些孤儿进程
```

#### 特点

- 孤儿进程会被 **init 进程（PID=1）** 收养
- init 进程会负责回收孤儿进程的资源（调用 `wait()`）
- 孤儿进程不会变成僵尸进程（因为 init 进程会及时回收）
- 孤儿进程可以继续正常运行，只是父进程变为 init

#### 与僵尸进程的区别

| 对比项 | 孤儿进程 | 僵尸进程 |
|-------|---------|---------|
| **定义** | 父进程退出，子进程仍在运行 | 进程退出，但父进程未调用 wait() 回收 |
| **状态** | 正常运行（PPID=1） | 已终止但未被回收（Z 状态） |
| **危害** | 无害，会被 init 收养 | 有害，占用进程表项和 PID |
| **处理** | 无需特殊处理 | 父进程调用 wait() 或被 init 收养 |

### 僵尸进程（Zombie Process）

#### 定义

进程已经退出（执行完毕或被杀死），但其父进程没有调用 `wait()` 或 `waitpid()` 回收其退出状态，导致进程在进程表中仍占用一个条目，这种进程就是**僵尸进程**。

#### 产生过程

```
父进程 fork() 创建子进程
       |
       v
子进程执行完毕，调用 exit() 退出
       |
       v
子进程的退出状态保存在内核中
       |
       v
父进程没有调用 wait() 回收
       |
       v
子进程成为僵尸进程（Z 状态）
```

#### 特点

- 僵尸进程已经终止，不再消耗 CPU 和内存
- 但僵尸进程在进程表中仍占用一个条目（PID）
- 如果大量僵尸进程积累，会耗尽可用的 PID，导致新进程无法创建
- 僵尸进程无法被 `kill` 命令杀死（因为它已经死了）

#### 如何处理僵尸进程

**1. 父进程调用 wait() 或 waitpid()**
```c
// 父进程回收子进程
int status;
pid_t pid = wait(&status);  // 阻塞等待子进程退出
// 或
pid_t pid = waitpid(child_pid, &status, WNOHANG);  // 非阻塞
```

**2. 使用信号处理**
```c
// 父进程注册 SIGCHLD 信号处理函数
signal(SIGCHLD, SIG_IGN);  // 忽略 SIGCHLD，内核自动回收
// 或
signal(SIGCHLD, sigchld_handler);  // 在 handler 中调用 wait()
```

**3. 双 fork 技巧（避免僵尸进程）**
```c
pid_t pid = fork();
if (pid == 0) {  // 子进程
    pid_t grandchild = fork();
    if (grandchild == 0) {  // 孙进程
        // 实际工作在这里完成
        do_work();
        exit(0);
    } else {  // 子进程立即退出
        exit(0);
    }
} else {  // 父进程
    wait(NULL);  // 回收子进程（不是孙进程）
    // 孙进程成为孤儿，被 init 收养，init 会回收它
}
```

**4. 查找僵尸进程**
```bash
ps aux | grep 'Z'            # 查找僵尸进程
ps -eo pid,ppid,stat,cmd | grep 'Z'  # 查看僵尸进程的 PID 和父进程 PID
```

### 用户态与内核态

#### 区别

| 对比项 | 用户态 | 内核态 |
|-------|--------|--------|
| **特权级别** | 低（Ring 3） | 高（Ring 0） |
| **可访问资源** | 只能访问被授权的资源 | 可以访问所有硬件和系统资源 |
| **可执行指令** | 只能执行非特权指令 | 可以执行所有指令（包括特权指令） |
| **内存访问** | 只能访问用户空间内存 | 可以访问用户空间和内核空间内存 |
| **运行代码** | 应用程序代码 | 操作系统内核代码 |
| **典型操作** | 计算、读写用户数据 | 系统调用、中断处理、设备驱动 |

#### 为什么需要区分用户态和内核态

1. **安全性**：防止应用程序直接操作硬件，避免系统崩溃
2. **稳定性**：一个应用程序的错误不会影响整个系统
3. **隔离性**：应用程序之间相互隔离，互不影响
4. **可控性**：操作系统可以控制应用程序的行为（如权限检查）

#### 切换方式

**1. 系统调用（System Call）**
- 应用程序主动请求内核服务
- 如：`read()`、`write()`、`open()`、`fork()`、`exec()`
- 过程：用户程序 → 中断（int 0x80/syscall） → 内核处理 → 返回用户程序

**2. 异常（Exception）**
- 程序执行过程中发生错误
- 如：缺页异常、除零错误、段错误
- 过程：CPU 检测到异常 → 内核处理 → 可能终止进程或修复错误

**3. 外部中断（Interrupt）**
- 外部设备请求 CPU 处理
- 如：键盘输入、网络数据到达、定时器中断
- 过程：硬件触发中断 → CPU 暂停当前程序 → 内核处理中断 → 恢复原程序

#### 切换过程

```
用户态程序执行
       |
       |（系统调用/异常/中断）
       v
保存用户态现场：
  - 程序计数器（PC）
  - 栈指针（SP）
  - 通用寄存器
  - 程序状态字（PSW）
       |
       v
切换到内核栈
       |
       v
执行内核代码
       |
       v
恢复用户态现场
       |
       v
切换回用户态，继续执行
```

### 进程调度算法

进程调度器决定哪个进程获得 CPU 时间。

#### 1. 先来先服务（FCFS，First Come First Served）

**原理**：按照进程到达的顺序，先到先执行。

**特点**：
- 实现简单，使用 FIFO 队列
- 非抢占式：进程一旦获得 CPU，就一直执行到完成
- **问题**：长作业在前时，短作业等待时间很长（**护航效应**）

**示例**：
```
进程  到达时间  执行时间
P1      0       24
P2      1       3
P3      2       3

执行顺序：P1 → P2 → P3
平均等待时间 = (0 + 23 + 24) / 3 = 15.67
```

#### 2. 短作业优先（SJF，Shortest Job First）

**原理**：选择执行时间最短的进程优先执行。

**特点**：
- 可以是抢占式（SRTF，最短剩余时间优先）或非抢占式
- **优点**：平均等待时间最短
- **缺点**：
  - 需要预知执行时间（实际中难以准确预测）
  - 可能导致长作业**饥饿**（一直得不到执行）

**示例**：
```
进程  到达时间  执行时间
P1      0       6
P2      2       2
P3      4       3

非抢占 SJF 执行顺序：P1 → P2 → P3
SRTF 执行顺序：P1(0-2) → P2(2-4) → P3(4-7) → P1(7-11)
```

#### 3. 时间片轮转（RR，Round Robin）

**原理**：每个进程轮流获得一个固定时间片（如 10ms），时间片用完后切换到下一个进程。

**特点**：
- 抢占式：时间片用完后强制切换
- **优点**：响应时间短，适合交互式系统
- **缺点**：时间片太大退化为 FCFS，太小上下文切换频繁
- **时间片选择**：通常 10~100ms

**示例**（时间片 = 4）：
```
进程  到达时间  执行时间
P1      0       10
P2      0       4
P3      0       6

执行顺序：P1(0-4) → P2(4-8) → P3(8-12) → P1(12-16) → P3(16-18) → P1(18-20)
```

#### 4. 优先级调度（Priority Scheduling）

**原理**：每个进程有一个优先级，优先级高的进程先执行。

**类型**：
- **静态优先级**：创建时确定，运行期间不变
- **动态优先级**：根据进程行为动态调整（如等待时间越长优先级越高）

**特点**：
- 可以是抢占式或非抢占式
- **问题**：低优先级进程可能**饥饿**
- **解决方案**：**老化（Aging）**——等待时间越长，优先级逐渐升高

#### 5. 多级反馈队列（Multilevel Feedback Queue）

**原理**：设置多个就绪队列，每个队列有不同的优先级和时间片。

```
高优先级队列（时间片小，如 8ms）
  └── 队列1：新进程首先进入这里
       |
       |（时间片用完但未完成）
       v
中优先级队列（时间片中，如 16ms）
  └── 队列2
       |
       |（时间片用完但未完成）
       v
低优先级队列（时间片大，如 32ms）
  └── 队列3：长作业最终会降到这个队列
```

**特点**：
- 结合了 FCFS、SJF、RR 的优点
- 短作业优先执行（在高优先级队列中很快完成）
- 长作业也能得到执行（最终降到低优先级队列）
- 交互式作业响应时间短
- 是大多数操作系统使用的调度算法（如 Linux 的 CFS 调度器）

**调度规则**：
1. 新进程进入最高优先级队列
2. 进程在当前队列的时间片内未完成，降到下一级队列
3. 高优先级队列为空时，才调度低优先级队列
4. 可以使用老化机制，防止低优先级队列饥饿

### Linux 进程状态

| 状态 | 含义 |
|------|------|
| **R（Running）** | 运行中或在运行队列中等待 |
| **S（Sleeping）** | 可中断睡眠（等待事件，可被信号唤醒） |
| **D（Disk Sleep）** | 不可中断睡眠（等待 IO，不能被信号唤醒） |
| **Z（Zombie）** | 僵尸进程（已终止但未被回收） |
| **T（Stopped）** | 被信号停止（如 Ctrl+Z） |
| **t（Tracing stop）** | 被调试器停止 |
| **I（Idle）** | 空闲（内核线程） |

**查看进程状态**：
```bash
ps aux | awk '{print $8}' | sort | uniq -c | sort -rn
# 或
ps -eo stat | sort | uniq -c | sort -rn
```
