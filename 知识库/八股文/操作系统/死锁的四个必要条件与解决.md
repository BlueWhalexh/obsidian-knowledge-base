# 死锁的四个必要条件与解决

## 问题
什么是死锁？死锁的四个必要条件是什么？如何预防、避免、检测和解除死锁？

## 答案

### 死锁的定义

死锁是指两个或多个进程在执行过程中，因互相等待对方持有的资源而无限期阻塞的现象。

### 四个必要条件

死锁的发生必须同时满足以下四个条件：

1. **互斥条件**：资源必须处于非共享模式，一次只有一个进程可以使用。其他进程申请该资源时必须等待
2. **占有并等待**：一个进程至少占有一个资源，并等待获取被其他进程占有的资源
3. **非抢占条件**：资源不能被抢占，只能在持有资源的进程主动释放后才能被其他进程获取
4. **循环等待**：存在一组等待进程 {P0, P1, ..., Pn}，P0等待P1占有的资源，P1等待P2占有的资源，...，Pn等待P0占有的资源

**关键**：只要破坏任一条件，死锁就不会发生。

### 预防死锁（破坏必要条件）

| 破坏条件 | 方法 |
|---------|------|
| 破坏互斥 | 使用可共享资源（实际中难以实现） |
| 破坏占有并等待 | 要求进程一次性申请所有需要的资源 |
| 破坏非抢占 | 设置优先级，允许高优先级进程抢占资源 |
| 破坏循环等待 | 对资源编号，按序申请资源 |

### 避免死锁

系统动态检测可能导致死锁的情况，避免进入死锁状态。

**银行家算法**：
- 每次资源申请时，判断分配后系统是否仍处于"安全状态"
- 如果资源分配会导致死锁，则拒绝该请求
- 安全状态：存在一个安全序列，能使所有进程都能顺利完成

### 检测死锁

允许死锁发生，但系统会定期检测并处理。

**死锁检测算法**：
- **资源分配图算法**：适用于单一资源类型，检测图中是否存在环路
- **等待图算法**：适用于多资源类型，检测进程间等待关系的环路

### 解除死锁

检测到死锁后，采取措施解除：

1. **终止进程**：选择一个或多个进程终止以打破死锁（选择代价最小的进程）
2. **回滚（Rollback）**：将进程回退到之前的某个安全状态重新执行
3. **资源抢占**：强行从某些进程中回收资源，重新分配给其他进程
