# 进程线程与协程

## 问题
进程、线程、协程有什么区别？它们之间的通信方式有哪些？进程间通信有哪些方式？线程间通信有哪些方式？

## 答案

### 进程、线程、协程对比

| 对比项 | 进程 | 线程 | 协程 |
|-------|------|------|------|
| **定义** | 系统资源分配的基本单位 | 进程内的执行单元 | 用户态的轻量级线程 |
| **内存** | 独立内存空间，互相隔离 | 共享进程的堆和全局变量，独立栈和寄存器 | 共享线程资源 |
| **开销** | 创建、切换、销毁开销大，需OS介入 | 切换开销中等，由OS调度 | 极低开销，用户态切换无内核介入 |
| **调度** | 操作系统调度 | 操作系统调度 | 程序自身控制 |
| **通信** | IPC（进程间通信） | 共享内存、锁等 | 直接访问共享变量 |
| **安全性** | 进程崩溃不影响其他进程 | 线程崩溃可能导致整个进程终止 | 协程崩溃影响所属线程 |

### 进程间通信方式（IPC）详解

#### 1. 管道（Pipe）

管道是最基本的 IPC 方式，本质是内核中的一个缓冲区。

**匿名管道（Anonymous Pipe）**：
- **特点**：半双工通信，数据只能单向流动
- **使用场景**：具有亲缘关系的进程之间（如父子进程）
- **实现原理**：通过 `pipe()` 系统调用创建，返回两个文件描述符，一个用于读（read end），一个用于写（write end）
- **典型应用**：Shell 中的管道命令 `ls | grep .txt`，将 ls 的输出作为 grep 的输入
- **生命周期**：随进程结束而销毁

```
父进程                     子进程
  写端 ───── 管道 ─────> 读端
```

**命名管道（Named Pipe / FIFO）**：
- **特点**：半双工通信，有文件系统路径名
- **使用场景**：无亲缘关系的进程之间
- **实现原理**：通过 `mkfifo()` 创建一个特殊文件，进程通过文件路径打开并读写
- **生命周期**：存在于文件系统中，显式删除或系统重启时销毁
- **阻塞行为**：读端打开时如果管道为空，会阻塞；写端打开时如果管道满，会阻塞

**管道的局限**：
- 数据只能单向流动（需要双向通信则需要两个管道）
- 缓冲区大小有限（通常 64KB）
- 只能传输无格式的字节流

#### 2. 消息队列（Message Queue）

消息队列是内核中的消息链表。

- **特点**：消息是有类型的、有格式的数据块
- **使用场景**：任意进程之间（不需要亲缘关系）
- **实现原理**：通过 `msgget()` 创建消息队列，`msgsnd()` 发送消息，`msgrcv()` 接收消息
- **消息类型**：每个消息都有一个类型字段，接收方可以按类型选择性接收
- **生命周期**：存在于内核中，显式删除或系统重启时销毁

**优点**：
- 发送方和接收方不需要同时在线
- 支持按类型选择性接收消息
- 避免了管道的命名和打开操作

**缺点**：
- 消息大小有限制（系统级限制，通常为 8KB~16KB）
- 每次消息传递都需要经过内核（用户空间 → 内核空间 → 用户空间），有数据拷贝开销
- 不适合大数据量传输

#### 3. 共享内存（Shared Memory）

共享内存是最快的 IPC 方式。

- **特点**：多个进程共享同一块物理内存区域
- **使用场景**：需要高性能数据交换的场景
- **实现原理**：
  - 通过 `shmget()` 创建共享内存段
  - 通过 `shmat()` 将共享内存映射到进程的地址空间
  - 进程直接读写共享内存，无需内核参与数据传输
- **生命周期**：存在于内核中，显式删除或系统重启时销毁

**优点**：
- **速度最快**：数据不需要在用户空间和内核空间之间拷贝
- 适合大数据量传输

**缺点**：
- **需要同步机制**：多个进程同时读写可能导致数据竞争，需要配合信号量或互斥锁使用
- **实现复杂**：需要处理同步、互斥、内存管理等问题

**使用流程**：
```
进程A                           进程B
  |                               |
  shmget() 创建共享内存            |
  shmat() 映射到地址空间            |
  |                               shmget() 获取同一共享内存
  |                               shmat() 映射到地址空间
  写入数据 ──> 共享内存 <── 读取数据
  |                               |
  shmdt() 脱离                    shmdt() 脱离
  shmctl() 删除                   |
```

#### 4. 信号量（Semaphore）

信号量是一个计数器，用于进程间的同步和互斥。

- **特点**：提供进程间的同步机制
- **使用场景**：控制多个进程对共享资源的访问
- **实现原理**：
  - 通过 `semget()` 创建信号量集
  - 通过 `semop()` 对信号量进行 P（等待/减）和 V（发送/加）操作
  - P 操作：信号量减 1，如果为 0 则阻塞
  - V 操作：信号量加 1，如果有阻塞的进程则唤醒

**信号量的两种用途**：

1. **互斥（Mutex）**：信号量初始值为 1，保证同一时刻只有一个进程访问临界区
```
semaphore = 1

进程A: P(sem) → 进入临界区 → V(sem)
进程B: P(sem) → 阻塞等待...
```

2. **同步（Synchronization）**：信号量初始值为 0，保证进程间的执行顺序
```
semaphore = 0

进程A: 先执行某操作 → V(sem)
进程B: P(sem) → 等待A完成后再执行
```

#### 5. 套接字（Socket）

套接字是最通用的 IPC 方式，可用于不同机器间的进程通信。

- **特点**：支持跨机器通信，支持多种协议
- **使用场景**：网络通信、不同机器间的进程通信
- **类型**：
  - **流式套接字（SOCK_STREAM）**：基于 TCP，可靠、有序、面向连接
  - **数据报套接字（SOCK_DGRAM）**：基于 UDP，无连接、不保证可靠
  - **原始套接字（SOCK_RAW）**：直接访问底层协议，用于网络编程

**Unix Domain Socket**：
- 用于同一机器上的进程间通信
- 不经过网络协议栈，比网络 Socket 更快
- 使用文件系统路径作为地址

**Socket 通信流程**：
```
服务端                           客户端
  socket() 创建                   socket() 创建
  bind() 绑定地址                 |
  listen() 监听                   |
  accept() 等待连接               connect() 发起连接
  |                               |
  read()/write() <──────────> read()/write()
  |                               |
  close() 关闭                    close() 关闭
```

#### IPC 方式对比总结

| 方式 | 传输方式 | 数据格式 | 性能 | 使用场景 |
|------|---------|---------|------|---------|
| 匿名管道 | 单向 | 字节流 | 中等 | 父子进程 |
| 命名管道 | 单向 | 字节流 | 中等 | 任意进程（同机器） |
| 消息队列 | 双向 | 消息块 | 中等 | 任意进程 |
| 共享内存 | 双向 | 任意 | 最快 | 大数据量、高性能场景 |
| 信号量 | 无数据传输 | 计数器 | 快 | 同步和互斥 |
| Socket | 双向 | 字节流/数据报 | 较慢 | 跨机器通信 |

### 线程间通信方式详解

线程共享同一进程的地址空间，因此线程间通信比进程间通信简单得多。

#### 1. 共享内存（共享变量）

线程共享进程的堆内存和全局变量，通过读写共享变量实现通信。

- **优点**：最直接、最高效的方式，无需系统调用
- **缺点**：需要同步机制（锁、原子操作）防止数据竞争
- **Java 中的实现**：
  - 共享对象（堆上的对象对所有线程可见）
  - `volatile` 关键字保证可见性
  - `synchronized` 关键字保证互斥和可见性
  - `Atomic*` 原子类（如 `AtomicInteger`）

**数据竞争示例**：
```java
// 多个线程同时执行 count++，可能出现数据不一致
// 因为 count++ 不是原子操作（读取→加1→写入）
private int count = 0;

// 解决方案1：synchronized
synchronized void increment() { count++; }

// 解决方案2：AtomicInteger
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet();
```

#### 2. 条件变量（Condition Variable）

条件变量用于线程间的等待/通知机制。

- **特点**：一个线程等待某个条件成立，另一个线程在条件成立时通知
- **使用场景**：生产者-消费者模式
- **Java 中的实现**：
  - `Object.wait()` / `Object.notify()` / `Object.notifyAll()`
  - `ReentrantLock` + `Condition`

**生产者-消费者示例**：
```java
// 使用 wait/notify
synchronized (queue) {
    while (queue.isEmpty()) {
        queue.wait();  // 消费者等待，释放锁
    }
    Object item = queue.remove();
    queue.notifyAll();  // 通知生产者
}

// 使用 Condition
lock.lock();
try {
    while (queue.isEmpty()) {
        notEmpty.await();  // 消费者等待
    }
    Object item = queue.remove();
    notFull.signal();  // 通知生产者
} finally {
    lock.unlock();
}
```

#### 3. 信号量（Semaphore）

信号量控制同时访问资源的线程数量。

- **特点**：计数器，限制并发访问的线程数
- **使用场景**：限流、资源池（如连接池、线程池）
- **Java 中的实现**：`java.util.concurrent.Semaphore`

```java
Semaphore semaphore = new Semaphore(10);  // 最多10个线程同时访问

// 获取许可
semaphore.acquire();  // 阻塞直到获取到许可
try {
    // 访问共享资源
} finally {
    semaphore.release();  // 释放许可
}
```

#### 4. 互斥锁（Mutex）

保证同一时刻只有一个线程访问临界区。

- **特点**：排他性访问
- **Java 中的实现**：
  - `synchronized` 关键字
  - `ReentrantLock`
  - `ReadWriteLock`（读写锁，读共享写排他）

```java
// 读写锁示例
ReadWriteLock rwLock = new ReentrantReadWriteLock();

// 读操作（多个线程可以同时读）
rwLock.readLock().lock();
try {
    // 读取数据
} finally {
    rwLock.readLock().unlock();
}

// 写操作（独占）
rwLock.writeLock().lock();
try {
    // 写入数据
} finally {
    rwLock.writeLock().unlock();
}
```

#### 线程间通信方式对比

| 方式 | 特点 | Java 实现 | 适用场景 |
|------|------|----------|---------|
| 共享变量 | 最直接、高效 | volatile、synchronized、Atomic | 数据共享 |
| 条件变量 | 等待/通知机制 | wait/notify、Condition | 生产者-消费者 |
| 信号量 | 控制并发数 | Semaphore | 限流、资源池 |
| 互斥锁 | 排他性访问 | synchronized、ReentrantLock | 临界区保护 |

### 协程的优势

- **极低开销**：切换在用户态完成，无需内核介入，切换时间在纳秒级
- **高并发**：单个线程可以运行成千上万个协程
- **无锁设计**：协程是协作式调度，避免了锁的开销

**适用场景**：IO密集型任务（如网络请求、文件读写），不适合CPU密集型任务。

**Java 中的协程**：
- Project Loom（Java 19+）引入了虚拟线程（Virtual Thread）
- 虚拟线程是 JVM 管理的轻量级线程，类似于协程
- `Thread.ofVirtual().start(() -> { ... })`
- `Executors.newVirtualThreadPerTaskExecutor()`

### 用户态与内核态

- **内核态**：操作系统运行在特权模式，可访问所有硬件和系统资源
- **用户态**：应用程序运行在非特权模式，只能访问被授权的资源
- 应用程序需要系统调用（如IO操作）时，会触发用户态到内核态的切换

**切换方式**：
1. **系统调用**：应用程序主动调用系统 API（如 read、write、open）
2. **异常**：应用程序发生异常（如缺页异常、除零错误）
3. **中断**：硬件设备触发中断（如磁盘读写完成、网络数据到达）
