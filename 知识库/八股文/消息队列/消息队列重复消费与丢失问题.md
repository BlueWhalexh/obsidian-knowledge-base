# 消息队列的优点及消息重复消费/丢失问题

## 问题
1. 消息队列有什么优点？
2. 如何保证消息不被重复消费？
3. 如何保证消息不丢失？

## 答案

### 消息队列三大优点

| 优点 | 说明 |
|------|------|
| **异步解耦** | 发消息就返回，不关心后续处理 |
| **削峰填谷** | 高峰期消息队列缓冲突发流量 |
| **扩展性** | 增加消费者就能提升处理能力 |

### 重复消费问题

**生产端**：开启幂等性校验（消息 ID 去重）

**消费端**：
- 用本地消息表记录已处理的消息 ID，消费前查表判断
- 有状态的任务用状态机控制（已处理就不能再处理）

### 消息丢失问题

分三段防护：

**生产端**：
- `acks=all`（所有副本确认）
- 失败重试
- 开启幂等性

**Broker**：
- 持久化到磁盘
- 配置刷盘策略
- 多副本同步

**消费端**：
- 关闭自动提交 offset
- 处理完业务再手动提交
