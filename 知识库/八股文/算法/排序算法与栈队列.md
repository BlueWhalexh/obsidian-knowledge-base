# 排序算法与栈队列

## 问题
归并排序和快速排序的原理是什么？它们的时间复杂度由什么决定？如何用两个栈模拟队列？

## 答案

### 排序算法对比总览

| 算法 | 平均时间 | 最坏时间 | 最好时间 | 空间 | 稳定性 |
|------|---------|---------|---------|------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(n) | O(1) | 稳定 |
| 选择排序 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 插入排序 | O(n²) | O(n²) | O(n) | O(1) | 稳定 |
| **归并排序** | O(nlogn) | O(nlogn) | O(nlogn) | **O(n)** | **稳定** |
| **快速排序** | O(nlogn) | O(n²) | O(nlogn) | O(logn) | **不稳定** |
| 堆排序 | O(nlogn) | O(nlogn) | O(nlogn) | O(1) | 不稳定 |
| 计数排序 | O(n+k) | O(n+k) | O(n+k) | O(k) | 稳定 |
| 基数排序 | O(d(n+k)) | O(d(n+k)) | O(d(n+k)) | O(n+k) | 稳定 |

### 归并排序（Merge Sort）

#### 基本思想

归并排序采用**分治法（Divide and Conquer）**：
1. **分（Divide）**：将数组从中间分成两个子数组
2. **治（Conquer）**：递归地对两个子数组进行归并排序
3. **合（Merge）**：将两个已排序的子数组合并成一个有序数组

#### 执行过程

```
原始数组：[38, 27, 43, 3, 9, 82, 10]

第1层分割：
[38, 27, 43, 3] 和 [9, 82, 10]

第2层分割：
[38, 27] [43, 3] 和 [9, 82] [10]

第3层分割：
[38] [27] [43] [3] [9] [82] [10]

开始合并（从底向上）：
[27, 38] [3, 43] [9, 82] [10]

继续合并：
[3, 27, 38, 43] [9, 10, 82]

最终合并：
[3, 9, 10, 27, 38, 43, 82]
```

#### 代码实现

```java
public class MergeSort {

    public static void mergeSort(int[] arr, int left, int right) {
        if (left >= right) return;

        int mid = left + (right - left) / 2;
        mergeSort(arr, left, mid);        // 递归排序左半部分
        mergeSort(arr, mid + 1, right);   // 递归排序右半部分
        merge(arr, left, mid, right);     // 合并两个有序部分
    }

    private static void merge(int[] arr, int left, int mid, int right) {
        int[] temp = new int[right - left + 1];
        int i = left;      // 左半部分指针
        int j = mid + 1;   // 右半部分指针
        int k = 0;         // 临时数组指针

        // 比较两个部分的元素，按顺序放入临时数组
        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j]) {
                temp[k++] = arr[i++];
            } else {
                temp[k++] = arr[j++];
            }
        }

        // 将左半部分剩余元素放入临时数组
        while (i <= mid) {
            temp[k++] = arr[i++];
        }

        // 将右半部分剩余元素放入临时数组
        while (j <= right) {
            temp[k++] = arr[j++];
        }

        // 将临时数组拷贝回原数组
        for (k = 0; k < temp.length; k++) {
            arr[left + k] = temp[k];
        }
    }

    public static void main(String[] args) {
        int[] arr = {38, 27, 43, 3, 9, 82, 10};
        mergeSort(arr, 0, arr.length - 1);
        System.out.println(Arrays.toString(arr));
        // 输出：[3, 9, 10, 27, 38, 43, 82]
    }
}
```

#### 复杂度分析

- **时间复杂度**：
  - 最好：O(nlogn) —— 每次都均匀分割
  - 最坏：O(nlogn) —— 每次都均匀分割
  - 平均：O(nlogn)
  - **无论数据分布如何，时间复杂度都是 O(nlogn)**

- **空间复杂度**：O(n) —— 合并时需要额外的临时数组
  - 递归调用栈深度为 O(logn)，但临时数组 O(n) 占主导

- **稳定性**：稳定 —— 合并时，相等元素的相对顺序保持不变

#### 为什么归并排序是稳定的？

在 `merge()` 函数中，当 `arr[i] <= arr[j]` 时，优先取左半部分的元素。这意味着相等的元素中，原来在前面的仍然在前面，保持了相对顺序。

### 快速排序（Quick Sort）

#### 基本思想

快速排序也采用**分治法**：
1. **选择基准（Pivot）**：从数组中选择一个元素作为基准
2. **分区（Partition）**：将数组分为两部分，左边都小于基准，右边都大于基准
3. **递归排序**：递归地对左右两部分进行快速排序

#### 执行过程

```
原始数组：[38, 27, 43, 3, 9, 82, 10]
选择基准：38

第1次分区：
小于38：[27, 3, 9, 10]
等于38：[38]
大于38：[43, 82]

递归处理左边：[27, 3, 9, 10]，基准=27
  小于27：[3, 9, 10]
  等于27：[27]
  大于27：[]
  递归处理 [3, 9, 10]，基准=3
    小于3：[]
    等于3：[3]
    大于3：[9, 10]
    递归处理 [9, 10]，基准=9
      小于9：[]
      等于9：[9]
      大于9：[10]

递归处理右边：[43, 82]，基准=43
  小于43：[]
  等于43：[43]
  大于43：[82]

最终结果：[3, 9, 10, 27, 38, 43, 82]
```

#### 代码实现

```java
public class QuickSort {

    public static void quickSort(int[] arr, int left, int right) {
        if (left >= right) return;

        int pivotIndex = partition(arr, left, right);
        quickSort(arr, left, pivotIndex - 1);   // 递归排序左边
        quickSort(arr, pivotIndex + 1, right);   // 递归排序右边
    }

    private static int partition(int[] arr, int left, int right) {
        // 选择基准（这里选择最右边的元素）
        int pivot = arr[right];
        int i = left;  // i 指向小于基准的区域的下一个位置

        for (int j = left; j < right; j++) {
            if (arr[j] < pivot) {
                swap(arr, i, j);
                i++;
            }
        }

        // 将基准放到正确位置
        swap(arr, i, right);
        return i;
    }

    private static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }

    public static void main(String[] args) {
        int[] arr = {38, 27, 43, 3, 9, 82, 10};
        quickSort(arr, 0, arr.length - 1);
        System.out.println(Arrays.toString(arr));
        // 输出：[3, 9, 10, 27, 38, 43, 82]
    }
}
```

#### 复杂度分析

- **时间复杂度**：
  - 最好：O(nlogn) —— 每次分区都均匀（基准恰好是中位数）
  - 最坏：O(n²) —— 每次分区都不均匀（基准是最大或最小值）
  - 平均：O(nlogn)

- **空间复杂度**：
  - 最好：O(logn) —— 递归调用栈深度
  - 最坏：O(n) —— 每次分区都不均匀，递归深度为 n

- **稳定性**：不稳定 —— 分区过程中可能改变相等元素的相对顺序

#### 快速排序时间复杂度由什么影响？

**1. 基准的选择**

基准的选择直接影响分区的均匀程度：
- **选到中位数**：分区均匀，时间复杂度 O(nlogn)
- **选到最大/最小值**：分区极度不均匀，时间复杂度退化为 O(n²)

**基准选择策略**：
- **固定选择第一个/最后一个**：最简单，但对有序数组会退化为 O(n²)
- **随机选择**：随机选择一个元素作为基准，期望时间复杂度 O(nlogn)
- **三数取中（Median of Three）**：取第一个、中间、最后一个三个数的中位数
- **九数取中**：从数组中取 9 个数，取中位数的中位数

**2. 数据分布**

- **随机数据**：平均情况，O(nlogn)
- **已排序数据**：如果选第一个/最后一个为基准，退化为 O(n²)
- **大量重复数据**：如果分区逻辑只分"小于"和"大于"，可能极度不均匀
  - 解决方案：**三路快排**（分为小于、等于、大于三部分）

**3. 三路快排（优化大量重复数据）**

```java
private static void quickSort3Way(int[] arr, int left, int right) {
    if (left >= right) return;

    int pivot = arr[left];
    int lt = left;      // arr[left..lt-1] < pivot
    int gt = right + 1;  // arr[gt..right] > pivot
    int i = left + 1;    // arr[lt..i-1] == pivot

    while (i < gt) {
        if (arr[i] < pivot) {
            swap(arr, lt, i);
            lt++;
            i++;
        } else if (arr[i] > pivot) {
            swap(arr, i, gt - 1);
            gt--;
        } else {
            i++;
        }
    }

    quickSort3Way(arr, left, lt - 1);
    quickSort3Way(arr, gt, right);
}
```

#### 归并排序 vs 快速排序

| 对比项 | 归并排序 | 快速排序 |
|-------|---------|---------|
| **时间复杂度** | 稳定 O(nlogn) | 平均 O(nlogn)，最坏 O(n²) |
| **空间复杂度** | O(n) | O(logn) |
| **稳定性** | 稳定 | 不稳定 |
| **原地排序** | 否（需要额外空间） | 是（只需递归栈） |
| **适用场景** | 链表排序、外部排序 | 数组排序（实践中更快） |
| **实际性能** | 稳定但常数因子较大 | 平均更快（缓存友好） |

**为什么实践中快速排序比归并排序快**？
1. **缓存友好**：快排是原地排序，数据访问的局部性好
2. **常数因子小**：快排的 swap 操作比归并的 merge 操作更简单
3. **空间优势**：快排只需 O(logn) 的栈空间

**什么时候用归并排序**？
1. **链表排序**：归并排序在链表上只需 O(1) 额外空间
2. **需要稳定性**：归并排序是稳定的
3. **外部排序**：数据量太大无法全部加载到内存时

### 用两个栈模拟队列

#### 思路

- 栈是后进先出（LIFO），队列是先进先出（FIFO）
- 使用两个栈：**入栈（stackIn）** 和 **出栈（stackOut）**
- **入队**：直接 push 到 stackIn
- **出队**：如果 stackOut 为空，将 stackIn 的所有元素 pop 并 push 到 stackOut，然后从 stackOut pop

#### 为什么这样做能模拟队列？

```
入队顺序：1, 2, 3

stackIn: [1, 2, 3]  ← push 顺序
stackOut: []

出队时，将 stackIn 倒入 stackOut：
stackIn: []
stackOut: [3, 2, 1]  ← pop 顺序（底部的 1 变成了栈顶）

从 stackOut pop：
stackOut.pop() = 1  ← 先进先出！
```

**关键点**：将元素从 stackIn 倒入 stackOut 后，元素的顺序被反转了，恰好变成了先进先出的顺序。

#### 代码实现

```java
public class TwoStackQueue<T> {
    private Stack<T> stackIn;   // 入栈
    private Stack<T> stackOut;  // 出栈

    public TwoStackQueue() {
        stackIn = new Stack<>();
        stackOut = new Stack<>();
    }

    // 入队
    public void enqueue(T item) {
        stackIn.push(item);
    }

    // 出队
    public T dequeue() {
        if (stackOut.isEmpty()) {
            if (stackIn.isEmpty()) {
                throw new RuntimeException("队列为空");
            }
            // 将 stackIn 的所有元素倒入 stackOut
            while (!stackIn.isEmpty()) {
                stackOut.push(stackIn.pop());
            }
        }
        return stackOut.pop();
    }

    // 查看队首元素
    public T peek() {
        if (stackOut.isEmpty()) {
            if (stackIn.isEmpty()) {
                throw new RuntimeException("队列为空");
            }
            while (!stackIn.isEmpty()) {
                stackOut.push(stackIn.pop());
            }
        }
        return stackOut.peek();
    }

    public boolean isEmpty() {
        return stackIn.isEmpty() && stackOut.isEmpty();
    }

    public static void main(String[] args) {
        TwoStackQueue<Integer> queue = new TwoStackQueue<>();
        queue.enqueue(1);
        queue.enqueue(2);
        queue.enqueue(3);
        System.out.println(queue.dequeue());  // 1（先进先出）
        System.out.println(queue.dequeue());  // 2
        queue.enqueue(4);
        System.out.println(queue.dequeue());  // 3
        System.out.println(queue.dequeue());  // 4
    }
}
```

#### 复杂度分析

- **入队**：O(1) —— 直接 push
- **出队**：
  - 最好：O(1) —— stackOut 不为空，直接 pop
  - 最坏：O(n) —— stackOut 为空，需要将 stackIn 的 n 个元素倒入 stackOut
  - **均摊**：O(1) —— 每个元素最多被 move 一次（从 stackIn 到 stackOut），所以平均还是 O(1)

#### 相关变体

**用两个队列模拟栈**：
- 使用两个队列 q1 和 q2
- **入栈**：将新元素加入非空的队列（如果都空，加入 q1）
- **出栈**：将非空队列中除最后一个元素外的所有元素转移到另一个队列，然后 pop 最后一个元素

```java
public class TwoQueueStack<T> {
    private Queue<T> q1;
    private Queue<T> q2;

    public TwoQueueStack() {
        q1 = new LinkedList<>();
        q2 = new LinkedList<>();
    }

    public void push(T item) {
        if (!q1.isEmpty()) {
            q1.offer(item);
        } else {
            q2.offer(item);
        }
    }

    public T pop() {
        if (q1.isEmpty() && q2.isEmpty()) {
            throw new RuntimeException("栈为空");
        }

        if (!q1.isEmpty()) {
            while (q1.size() > 1) {
                q2.offer(q1.poll());
            }
            return q1.poll();
        } else {
            while (q2.size() > 1) {
                q1.offer(q2.poll());
            }
            return q2.poll();
        }
    }
}
```

### 补充：堆排序

#### 基本思想

利用**堆（Heap）** 这种数据结构进行排序：
1. **建堆**：将数组构建成大顶堆（升序）或小顶堆（降序）
2. **排序**：将堆顶元素（最大值）与末尾元素交换，然后调整堆，重复此过程

```java
public class HeapSort {

    public static void heapSort(int[] arr) {
        int n = arr.length;

        // 建堆（从最后一个非叶子节点开始）
        for (int i = n / 2 - 1; i >= 0; i--) {
            heapify(arr, n, i);
        }

        // 排序
        for (int i = n - 1; i > 0; i--) {
            swap(arr, 0, i);       // 将堆顶（最大值）交换到末尾
            heapify(arr, i, 0);    // 重新调整堆
        }
    }

    // 调整堆（下沉操作）
    private static void heapify(int[] arr, int n, int i) {
        int largest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;

        if (left < n && arr[left] > arr[largest]) {
            largest = left;
        }
        if (right < n && arr[right] > arr[largest]) {
            largest = right;
        }

        if (largest != i) {
            swap(arr, i, largest);
            heapify(arr, n, largest);  // 递归调整
        }
    }

    private static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

#### 复杂度

- 时间：O(nlogn)（建堆 O(n)，排序 O(nlogn)）
- 空间：O(1)（原地排序）
- 稳定性：不稳定
