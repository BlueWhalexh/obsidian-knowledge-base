# HTTP 状态码

## 问题
HTTP 常见的状态码有哪些？分别代表什么意思？在什么场景下使用？

## 答案

HTTP 状态码由三位数字组成，第一个数字定义了响应的类别：

| 类别 | 范围 | 含义 |
|------|------|------|
| 1xx | 100-199 | 信息性状态码，表示请求已被接受，需要继续处理 |
| 2xx | 200-299 | 成功，请求被成功接收、理解、处理 |
| 3xx | 300-399 | 重定向，需要进一步操作才能完成请求 |
| 4xx | 400-499 | 客户端错误，请求包含语法错误或无法完成 |
| 5xx | 500-599 | 服务端错误，服务器在处理请求时发生错误 |

---

### 2xx 成功

**200 OK**
- **含义**：请求成功，服务器已返回请求的资源
- **使用场景**：GET 请求返回资源、POST 请求成功处理
- **响应体**：包含请求的资源内容

**204 No Content**
- **含义**：请求成功，但响应体中没有内容
- **使用场景**：DELETE 请求成功删除资源、PUT 请求更新成功但不需要返回内容
- **响应体**：无
- **特点**：浏览器收到 204 后不会刷新页面

**206 Partial Content**
- **含义**：服务器成功处理了部分 GET 请求（范围请求）
- **使用场景**：断点续传、视频拖动播放、大文件分段下载
- **请求头**：客户端通过 `Range` 头部指定要获取的字节范围
- **响应头**：服务器通过 `Content-Range` 头部返回实际传输的范围
- **示例**：
```
# 客户端请求
GET /video.mp4 HTTP/1.1
Range: bytes=0-1023

# 服务器响应
HTTP/1.1 206 Partial Content
Content-Range: bytes 0-1023/5000
Content-Length: 1024
```

### 3xx 重定向

**301 Moved Permanently（永久重定向）**
- **含义**：请求的资源已永久移动到新的 URL
- **使用场景**：网站更换域名、URL 结构调整
- **特点**：
  - 浏览器会自动跳转到新 URL
  - 浏览器会缓存重定向，后续请求直接访问新 URL
  - 搜索引擎会更新索引，将旧 URL 的权重转移到新 URL
- **与 302 的区别**：301 是永久性的，302 是临时性的

**302 Found（临时重定向）**
- **含义**：请求的资源临时从不同的 URL 响应
- **使用场景**：未登录时跳转到登录页面、维护页面临时跳转
- **特点**：
  - 浏览器会自动跳转到新 URL
  - 浏览器不会缓存重定向，每次请求仍访问原 URL
  - 搜索引擎不会更新索引
- **注意**：HTTP 1.0 中使用 302，HTTP 1.1 规范推荐使用 303 和 307 替代

**303 See Other**
- **含义**：请求的资源可以在另一个 URL 找到，应使用 GET 方法获取
- **使用场景**：POST 请求后的重定向（PRG 模式：Post/Redirect/Get）
- **特点**：强制客户端使用 GET 方法访问新 URL

**304 Not Modified（未修改）**
- **含义**：资源未修改，客户端可以使用缓存的版本
- **使用场景**：协商缓存命中
- **响应头**：不包含响应体，只包含更新的头部信息（如新的过期时间）
- **协商缓存流程**：
  1. 客户端首次请求资源，服务器返回资源和 `Last-Modified` / `ETag`
  2. 客户端再次请求时，在请求头中携带 `If-Modified-Since` / `If-None-Match`
  3. 服务器检查资源是否修改：
     - 未修改 → 返回 304，客户端使用缓存
     - 已修改 → 返回 200 和新资源

**307 Temporary Redirect（临时重定向）**
- **含义**：与 302 类似，但**严格要求客户端不改变请求方法**
- **使用场景**：需要保持 POST 方法的临时重定向
- **与 302 的区别**：302 在某些浏览器中会将 POST 变为 GET，307 不会

**308 Permanent Redirect（永久重定向）**
- **含义**：与 301 类似，但**严格要求客户端不改变请求方法**
- **使用场景**：需要保持 POST 方法的永久重定向

### 4xx 客户端错误

**400 Bad Request**
- **含义**：请求语法错误，服务器无法理解
- **使用场景**：
  - 请求参数格式错误（如 JSON 解析失败）
  - 请求头格式错误
  - 缺少必要参数
- **示例**：`{"error": "Invalid JSON format"}`

**401 Unauthorized**
- **含义**：请求需要身份验证（未认证）
- **使用场景**：未登录或 Token 过期
- **响应头**：必须包含 `WWW-Authenticate` 头部，告知客户端如何进行身份验证
- **与 403 的区别**：401 表示"未认证"（可以重新登录），403 表示"已认证但无权限"

**403 Forbidden**
- **含义**：服务器理解请求，但拒绝执行（无权限）
- **使用场景**：
  - 已登录但没有访问权限
  - IP 被禁止访问
  - 文件权限不足
- **特点**：即使重新认证也不会改变结果

**404 Not Found**
- **含义**：请求的资源不存在
- **使用场景**：
  - URL 输入错误
  - 资源已被删除
  - 路由配置错误
- **特点**：最常见的 HTTP 错误之一

**405 Method Not Allowed**
- **含义**：请求方法不被允许
- **使用场景**：使用了服务器不支持的 HTTP 方法（如对只支持 GET 的接口发送 POST）
- **响应头**：必须包含 `Allow` 头部，告知该资源支持哪些 HTTP 方法

**408 Request Timeout**
- **含义**：服务器等待请求超时
- **使用场景**：客户端发送请求太慢，服务器主动断开连接

**409 Conflict**
- **含义**：请求与服务器当前状态冲突
- **使用场景**：并发修改同一资源导致冲突（如乐观锁冲突）

**413 Payload Too Large**
- **含义**：请求体过大，服务器无法处理
- **使用场景**：上传文件超过服务器限制

**429 Too Many Requests**
- **含义**：客户端发送请求过于频繁（限流）
- **使用场景**：API 限流、防刷保护
- **响应头**：通常包含 `Retry-After` 头部，告知客户端多久后可以重试

### 5xx 服务端错误

**500 Internal Server Error**
- **含义**：服务器内部错误，无法完成请求
- **使用场景**：
  - 代码 Bug（空指针、数组越界等）
  - 数据库连接失败
  - 服务器配置错误
- **特点**：最常见的服务端错误，通常需要查看服务器日志定位问题

**502 Bad Gateway**
- **含义**：网关或代理服务器从上游服务器收到无效响应
- **使用场景**：
  - Nginx 反向代理后端服务不可用
  - 后端服务崩溃或超时
  - 上游服务器返回了无效的响应
- **典型场景**：Nginx → Tomcat，Tomcat 挂了返回 502

**503 Service Unavailable**
- **含义**：服务器暂时无法处理请求（过载或维护）
- **使用场景**：
  - 服务器正在维护
  - 服务器过载（如大促流量高峰）
  - 应用重启中
- **响应头**：通常包含 `Retry-After` 头部，告知客户端多久后重试
- **与 500 的区别**：503 是暂时性的（过会儿可能恢复），500 是服务器代码错误

**504 Gateway Timeout**
- **含义**：网关或代理服务器等待上游服务器响应超时
- **使用场景**：
  - Nginx 反向代理后端服务响应太慢
  - 后端服务处理时间超过了网关的超时时间
- **与 502 的区别**：502 是收到无效响应，504 是没有收到响应（超时）

### 状态码速查表

| 状态码 | 含义 | 典型场景 |
|-------|------|---------|
| 200 | OK | 请求成功 |
| 204 | No Content | DELETE 成功 |
| 206 | Partial Content | 断点续传 |
| 301 | Moved Permanently | 域名更换 |
| 302 | Found | 临时跳转 |
| 304 | Not Modified | 缓存命中 |
| 400 | Bad Request | 参数错误 |
| 401 | Unauthorized | 未登录 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 405 | Method Not Allowed | 方法不支持 |
| 429 | Too Many Requests | 限流 |
| 500 | Internal Server Error | 代码 Bug |
| 502 | Bad Gateway | 后端挂了 |
| 503 | Service Unavailable | 服务过载/维护 |
| 504 | Gateway Timeout | 后端超时 |
