# 从输入URL到页面显示发生了什么

## 问题
从用户在浏览器输入URL到页面完整显示，整个过程中发生了什么？

## 答案

### 完整流程概览

```
输入URL → DNS解析 → TCP连接 → TLS握手 → 发送HTTP请求 → 服务器处理 → 返回响应 → 浏览器渲染
```

### 1. DNS解析

将域名转换为服务器IP地址的过程：
- 浏览器缓存 → 系统缓存 → 路由器缓存 → DNS服务器递归查询
- 最终获取到服务器的IP地址

### 2. 建立TCP连接

通过三次握手与服务器建立TCP连接（详见[[TCP 三次握手与四次挥手]]）。

### 3. TLS握手（HTTPS场景）

如果是HTTPS，还需要进行TLS握手：
- 协商加密算法
- 验证服务器证书
- 交换密钥
- 生成会话密钥

### 4. 发送HTTP请求

客户端构建HTTP请求报文，包含：
- 请求行（方法、URL、协议版本）
- 请求头（Host、Cookie、User-Agent等）
- 请求体（POST请求的数据）

### 5. 服务器处理请求

- 负载均衡：DNS负载均衡 → 网络层负载均衡（LVS） → 应用层负载均衡（Nginx）
- Web服务器接收请求，路由到对应的处理程序
- 可能涉及数据库查询、缓存读取、业务逻辑处理

### 6. 返回HTTP响应

服务器返回响应报文：
- 状态行（状态码：200、301、404、500等）
- 响应头（Content-Type、Set-Cookie等）
- 响应体（HTML、JSON等）

### 7. 浏览器渲染

**渲染流程**：

1. **构建DOM树**：解析HTML文档，生成DOM（Document Object Model）树
2. **构建CSSOM树**：解析CSS，生成CSSOM（CSS Object Model）树
3. **构建渲染树**：将DOM树和CSSOM树合并，生成渲染树（Render Tree）。不可见的元素（如`display: none`）不会出现在渲染树中
4. **布局（Layout/Reflow）**：计算渲染树中每个节点的位置和大小
5. **绘制（Paint）**：将渲染树的节点转换为屏幕上的实际像素
6. **合成（Composite）**：将不同的图层合成最终页面

**优化相关**：
- CSS放在`<head>`中，避免FOUC（无样式内容闪烁）
- JS放在`</body>`前，避免阻塞DOM解析
- 图片懒加载，减少首屏加载时间

### CDN的作用

CDN（Content Delivery Network）将静态资源缓存到全球各地的边缘节点：
- 用户请求时，从最近的节点获取资源，减少延迟
- 减轻源站压力
- 提高可用性和容错能力
